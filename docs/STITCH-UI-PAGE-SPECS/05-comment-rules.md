# 留言中心 — 規則與模板

## 1. 頁面用途
- **做什麼**：設定自動規則、模板與商品對應、風險與導流；含**留言規則測試器**驗證規則命中。
- **給誰**：`super_admin`、`marketing_manager`、`cs_agent`。
- **情境**：建規則、調模板、測試命中、管理風險導流行為。

## 2. 進入方式
- **route**：`/comment-center/rules`。
- **從哪進**：側欄「留言中心」→ Tab「規則與模板」。
- **權限**：全員可進；實際編輯權限若有細分 **待確認**。

## 3. 首屏必須看到的內容
1. **子 Tab 列**：自動規則／模板與商品對應／風險與導流（及內容區切換）。
2. **規則相關**：五桶（或分類）說明區（INVENTORY 已述「五桶」）— **具體標題以程式為準**。
3. **留言規則測試器**：獨立區塊標題、輸入/選擇、**測試**按鈕、結果區。
4. **列表或表單**：規則/模板條目與操作（新增、編輯、啟停等 **待確認**）。
5. **儲存/套用**類主線（若為表單頁）— **待確認**。

## 4. 區塊排列順序
- 頂：子 Tab。
- 內容依 Tab：規則列表+說明 → 測試器（建議測試器與規則編輯同頁可見或同 Tab 內可捲動到）。
- 其餘 Tab：模板與商品對應、風險與導流各自獨立區塊。

## 5. 按鈕盤點
### A. 主按鈕
- **規則測試器「測試」**（或同等文案）：執行命中並顯示結果。
- **儲存規則/模板**（若有）— **待確認**。
### B. 次按鈕
- 新增規則、編輯、複製、啟用/停用— **待確認**。
### C. 危險
- 刪除規則、大量套用— **待確認**是否需二次確認。
### D. 輔助
- 說明折疊、範例展開— **待確認**。

## 6. 篩選 / 控制
- 規則列表可能含搜尋、品牌、狀態— **待確認**。
- 測試器輸入欄位（留言內容、來源、商品等）— **以 UI 為準**。

## 7. 狀態 / badge / warning
- 規則啟停、命中結果 success/error 區塊。
- 風險相關警示 copy— **待確認**。

## 8. 可收合但不能消失
- 子 Tab 三項不可減為單頁無標籤（除非改路由並更新 INVENTORY）。
- 測試器區塊不可移除（驗收與除錯核心）。

## 9. 最不能被 AI 誤改
- 測試器與結果顯示（避免「美化」成無輸出區）。
- 五桶/分類語意與導流風險說明。

## 10. Stitch 用摘要
規則與模板頁含三個子 Tab：自動規則、模板與商品對應、風險與導流。自動規則區需保留規則列表與分類說明，以及「留言規則測試器」— 含輸入、測試按鈕、結果區，為除錯核心。主 CTA 為測試與儲存類（若有）。視覺可 Tab 樣式與間距優化，但不可刪子 Tab、不可把測試器改成裝飾區、不可隱藏命中結果。其餘按鈕細節以實作為準。
