# 留言中心 — 批次建頁（管理員）

## 1. 頁面用途
- **做什麼**：以長效 Token 取得粉專列表、勾選後指定品牌、一鍵建立/綁定相關頁面或設定（批次作業）。
- **給誰**：**僅 `super_admin`**（INVENTORY 依 `ROUTE_ACCESS` / 條件渲染）。
- **情境**：多粉專一次建置、減少重複手動綁定。

## 2. 進入方式
- **route**：`/comment-center/batch-pages`。
- **從哪進**：側欄「留言中心」內 Tab 或選單（**僅管理員可見**）。
- **權限**：非 super_admin 不應看到入口；若直連 URL 應權限不足頁— **待確認**行為。

## 3. 首屏必須看到的內容
1. **Token 輸入區**：貼上、驗證或取得列表前提示。
2. **「取得粉專」**（或同等）觸發後載入列表。
3. **勾選列表**：多選粉專/頁面。
4. **品牌選擇**：下拉或對照指定品牌。
5. **一鍵建立**主按鈕與進度/結果摘要。

## 4. 區塊排列順序
- Token → 取得 → 列表（可捲動）→ 品牌 → 執行批次 → 結果 log。

## 5. 按鈕盤點
### A. 主按鈕
- **取得粉專列表**、**一鍵建立**（或「開始批次」）。
### B. 次按鈕
- 全選/取消全選、清除 Token— **待確認**。
### C. 危險
- 覆寫既有綁定— **待確認**是否有紅色確認。
### D. 輔助
- 說明連結、權限範圍提示。

## 6. 篩選 / 控制
- 列表內搜尋粉專名— **待確認**。
- 品牌必填驗證。

## 7. 狀態 / badge / warning
- Token 無效、API 失敗錯誤。
- 批次進度條或逐步結果— **待確認**。

## 8. 可收合但不能消失
- Token 與批次執行區不可「精簡」成無輸入即送出。

## 9. 最不能被 AI 誤改
- **僅 super_admin** 可見與可操作（資安）。
- 勾選 + 品牌對照再執行（防誤觸全站）。

## 10. Stitch 用摘要
批次建頁為管理員專用：輸入長效 Token、取得粉專列表、勾選多筆、選品牌、一鍵批次建立。主 CTA 為取得列表與執行批次。視覺可 wizard 化，但不可對非管理員顯示、不可隱藏品牌選擇、不可無確認即批次。錯誤與結果須可讀。細節以 `comment-center` 實作為準。
