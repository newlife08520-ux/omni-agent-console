# 績效儀表板

## 1. 頁面用途
- **做什麼**：檢視客服或團隊績效指標（回覆時間、處理量、評分等— **依 `performance.tsx`**）。
- **給誰**：`super_admin`、`marketing_manager`（**待確認**）。
- **情境**：排班檢討、個人/團隊對比。

## 2. 進入方式
- **route**：`/performance`。
- **從哪進**：側欄「績效」或同等文案。
- **權限**：依路由設定。

## 3. 首屏必須看到的內容
1. **頁面標題**。
2. **時間或成員範圍選擇**。
3. **核心 KPI 列**（至少 2–4 指標）。
4. **明細表或排行**— **待確認**。
5. **載入/錯誤狀態**。

## 4. 區塊排列順序
- 篩選 → KPI → 圖表/表 → 明細— **以實作為準**。

## 5. 按鈕盤點
### A. 主按鈕
- **套用**、**匯出報表**— **待確認**。
### B. 次按鈕
- 切換個人/團隊視角— **待確認**。
### C. 危險
- 通常無。
### D. 輔助
- Tooltip 指標定義— **待確認**。

## 6. 篩選 / 控制
- 日期、成員、品牌、渠道— **待確認**。

## 7. 狀態 / badge / warning
- 資料不足、權限不足區塊— **待確認**。

## 8. 可收合但不能消失
- 指標定義不可完全移除（可 ? 圖示）。

## 9. 最不能被 AI 誤改
- 個資相關：成員名單顯示範圍需符合角色— **待確認**。

## 10. Stitch 用摘要
績效頁與分析頁類似但更偏人員/處理效率。首屏需篩選與 KPI，並保留明細或排行區。主 CTA 為套用與匯出（若有）。視覺可與分析頁共用元件風格，但不可刪篩選、不可讓指標無說明。細節以 `performance.tsx` 為準。
