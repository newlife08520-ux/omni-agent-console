# 留言中心 — 模擬測試

## 1. 頁面用途
- **做什麼**：建立測試留言、模擬 Webhook 事件、一鍵產生多筆測試資料，驗證規則與收件匣行為。
- **給誰**：`super_admin`、`marketing_manager`、`cs_agent`。
- **情境**：上線前驗收、重現客訴情境、壓力前小批量測試。

## 2. 進入方式
- **route**：`/comment-center/simulate`。
- **從哪進**：側欄「留言中心」→ Tab「模擬測試」或同等（以程式為準）。
- **權限**：全員可進；若僅特定角色 **待確認**。

## 3. 首屏必須看到的內容
1. **測試留言建立器**：欄位（內容、來源、使用者資訊等）與**送出建立**。
2. **模擬 Webhook**：URL/載荷或預設範本與**觸發**。
3. **一鍵 6 筆**（或程式內固定 N 筆）：批次建立捷徑與結果提示。
4. **結果/日誌區**：最近一次成功或失敗訊息。
5. **警告 copy**：僅測試環境或資料影響說明— **待確認**是否有。

## 4. 區塊排列順序
- 上→下：建立器 Card → Webhook Card → 批次捷徑 → 結果區（或可並排兩欄 **待確認** RWD）。

## 5. 按鈕盤點
### A. 主按鈕
- **建立測試留言**、**發送 Webhook**、**一鍵建立 N 筆**（三條主線至少各一顆 primary 或語意等同）。
### B. 次按鈕
- 清除表單、載入範例— **待確認**。
### C. 危險
- 清空測試資料— **待確認**。
### D. 輔助
- 複製 payload、開文件— **待確認**。

## 6. 篩選 / 控制
- 品牌/渠道選擇（若建立器需指定）— **待確認**。
- JSON 編輯 vs 表單— **待確認**。

## 7. 狀態 / badge / warning
- API 錯誤 toast 或 inline alert。
- 成功計數— **待確認**。

## 8. 可收合但不能消失
- 三條主線（單筆、Webhook、批次）不可合併到只剩一種而無說明。

## 9. 最不能被 AI 誤改
- Webhook 與建立器並存（整合測試依賴）。
- 一鍵批次按鈕（回歸測試效率）。

## 10. Stitch 用摘要
模擬測試頁提供三塊：測試留言表單+建立、模擬 Webhook 觸發、一鍵多筆建立。每塊需有明確主按鈕與結果回饋。版面可改為兩欄卡片，但三種測試路徑都須首屏或一捲內可達。不可刪 Webhook 或批次捷徑、不可把結果區改成無文字狀態。文案與欄位以實作為準。
