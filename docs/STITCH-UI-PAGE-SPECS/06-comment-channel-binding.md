# 留言中心 — 渠道綁定（粉專與 LINE）

## 1. 頁面用途
- **做什麼**：檢視與管理 Facebook/Instagram/LINE 等渠道與品牌、Webhook、導向設定；九欄表呈現綁定狀態。
- **給誰**：`super_admin`、`marketing_manager`、`cs_agent`。
- **情境**：上線綁定、核對 channel id、排查未綁定列。

## 2. 進入方式
- **route**：`/comment-center/channel-binding`。
- **從哪進**：側欄「留言中心」→ Tab「渠道綁定」或同等文案（以 `comment-center.tsx` Tabs 為準）。
- **權限**：全員可進；敏感操作若有 **待確認**。

## 3. 首屏必須看到的內容
1. **頁面標題/說明**：渠道綁定用途簡述。
2. **九欄表**（INVENTORY 所述）：平台、顯示名稱、Channel ID、品牌、狀態、Webhook、最後同步、操作等— **欄位名以程式為準**。
3. **篩選或搜尋**（若有）：品牌、平台— **待確認**。
4. **操作欄**：編輯、重新綁定、複製連結等— **待確認**。
5. **空狀態/載入**：無資料時說明。

## 4. 區塊排列順序
- 頂：標題與可選工具列。
- 中：主表（橫向捲動若欄多）。
- 底：分頁或「載入更多」— **待確認**。

## 5. 按鈕盤點
### A. 主按鈕
- **儲存綁定**（Dialog 或行內）— **待確認**。
### B. 次按鈕
- 新增綁定、重新整理、匯出— **待確認**。
### C. 危險
- 解除綁定、刪除渠道— **待確認**是否二次確認。
### D. 輔助
- 複製 Channel ID、開外部說明— **待確認**。

## 6. 篩選 / 控制
- 品牌 scope 可能繼承側欄— **待確認**與表內篩選關係。
- 平台 Tab 或下拉— **待確認**。

## 7. 狀態 / badge / warning
- 已綁定/未綁定/錯誤態色票。
- Webhook 健康或最後同步時間— **待確認**顯示格式。

## 8. 可收合但不能消失
- 表頭與九欄語意不可縮成「僅圖示」而無標籤。
- Channel ID 與品牌對照欄不可刪。

## 9. 最不能被 AI 誤改
- 技術識別碼（Channel ID）可讀性與複製路徑。
- 綁定狀態與 Webhook 欄（除錯）。

## 10. Stitch 用摘要
渠道綁定為留言中心子頁，主體為寬表（約九欄）列出各渠道與品牌、狀態、Webhook、同步與操作。首屏須能掃描多列綁定狀態。主 CTA 為儲存/綁定類；次為重新整理、複製 ID。視覺可做 sticky 表頭與 zebra，但不可刪欄導致無法對照品牌與 Channel ID、不可把狀態改成無色階。細節按鈕以實作為準。
