# 留言中心 — 收件匣

## 1. 頁面用途
- **做什麼**：瀏覽、篩選、檢視留言與回覆狀態；手動回覆、指派、標記、觸發 AI 等。
- **給誰**：`super_admin`、`marketing_manager`、`cs_agent`。
- **情境**：日常處理留言、依狀態/來源/品牌篩選、看詳情與歷史。

## 2. 進入方式
- **route**：`/comment-center/inbox`。
- **從哪進**：側欄「留言中心」→ 預設 Tab「收件匣」。
- **權限**：全員可進。

## 3. 首屏必須看到的內容
1. **失敗告警橫條**（`failedCommentsCount > 0`）：顯示筆數與「查看失敗留言」。
2. **錯誤原因摘要**（`errorReasons.length > 0`）：可點「查看詳情」開 Dialog。
3. **健康九宮格**（`HEALTH_METRICS`）：9 格 KPI，含「戰情摘要」可點數字開 Dialog。
4. **規則命中／完成分布**（`RULE_HIT_METRICS` + `COMPLETION_METRICS`）：多列統計。
5. **篩選列**：搜尋、狀態、來源、品牌、日期、指派、排序、重新整理、**匯出 CSV**。
6. **主體**：左列表 + 右詳情（或空狀態）。

## 4. 區塊排列順序
- 頂→下：失敗條 → 錯誤條 → 九宮格 → 規則/完成列 → 篩選 Card → 左右分欄（列表 min 360px / 詳情 flex-1）。

## 5. 按鈕盤點
### A. 主按鈕
- **「發送」**（詳情內回覆 composer）：送出留言回覆。
- **「查看失敗留言」**：導向同頁並帶 `status=failed`（**待確認**實際 query 鍵名）。
### B. 次按鈕
- 「查看詳情」（錯誤 Dialog）、九宮格內「戰情摘要」觸發、篩選內「重新整理」「匯出 CSV」、詳情內「指派」「標記已讀」「觸發 AI」等（**詳盡列表待確認** `comment-center.tsx`）。
### C. 危險
- 刪除/拒絕類（若有）— **待確認**。
### D. 輔助
- 列表列點選、分頁或無限捲動控制（**待確認**）。

## 6. 篩選 / 控制
- **搜尋**：關鍵字。
- **狀態**：`STATUS_OPTIONS`（全部、待處理、已回覆、失敗等）。
- **來源**：`SOURCE_OPTIONS`（全部、Facebook、Instagram、LINE）。
- **品牌**：下拉（含全部）。
- **日期**：起迄。
- **指派**：篩選對象。
- **排序**：欄位 + 升降冪。

## 7. 狀態 / badge / warning
- 列表/詳情：`待補資料` 等語意標籤。
- 失敗/錯誤原因用色須明顯。
- 統計格數字可點（戰情摘要）— **不可**改成純裝飾。

## 8. 可收合但不能消失
- Dialog 可關；**九宮格與篩選列**不可整塊移除。
- 左右分欄在桌面須可並列（可 RWD 改上下但須同頁可達）。

## 9. 最不能被 AI 誤改
- 失敗與錯誤入口（營運除錯）。
- 匯出 CSV 與品牌/狀態語意。
- 「待補資料」與指派流程。

## 10. Stitch 用摘要
收件匣頂部為營運健康：失敗留言橫條、錯誤原因摘要、九宮格 KPI（含可點戰情摘要）、規則命中與完成分布；其下為完整篩選列與匯出。主區左列表右詳情，詳情內發送回覆為主 CTA。視覺可卡片化與精簡統計密度，但不可刪失敗/錯誤入口、不可拿掉匯出與狀態來源篩選、不可讓詳情無法手動回覆。桌面維持雙欄語意。
