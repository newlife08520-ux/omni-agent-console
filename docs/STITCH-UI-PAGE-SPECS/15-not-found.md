# 404 找不到頁面

## 1. 頁面用途
- **做什麼**：告知使用者 URL 不存在或無權限路由錯誤時的 fallback（依 `App.tsx` Switch default）。
- **給誰**：任何已載入應用者。
- **情境**：書籤失效、打錯路徑。

## 2. 進入方式
- **route**：未匹配路徑（如任意未定義 path）。
- **從哪進**：直接輸入錯誤 URL。
- **權限**：無特別；仍可能在 shell 內— **待確認** 404 是否包在 `AuthenticatedApp`。

## 3. 首屏必須看到的內容
1. **404 或「找不到頁面」標題**。
2. **簡短說明**。
3. **返回首頁或上一頁**連結/按鈕。
4. （可選）插圖— 不可取代文字訊息。
5. **側欄是否保留**— **待確認**；若保留則主內容區仍須有上述三點。

## 4. 區塊排列順序
- 垂直置中或主內容區塊：標題 → 說明 → 按鈕。

## 5. 按鈕盤點
### A. 主按鈕
- **回首頁** `Link` 或 `button` 導向 `/`。
### B. 次按鈕
- **返回**（history back）— **待確認**。
### C. 危險
- 無。
### D. 輔助
- 聯絡管理員— **待確認**。

## 6. 篩選 / 控制
- 無。

## 7. 狀態 / badge / warning
- 無動態狀態；可靜態提示「若您認為這是錯誤…」。

## 8. 可收合但不能消失
- 返回路徑不可只剩關閉分頁暗示。

## 9. 最不能被 AI 誤改
- 不可改成空白頁或無限 loading。
- 首頁連結必須可點且明確。

## 10. Stitch 用摘要
404 為簡潔訊息頁：標題、一句說明、主按鈕回首頁（可次選返回）。可插圖但不可無文字。若在 shell 內，內容限於 main 區。視覺可插畫與柔和色，但不可移除 CTA、不可全螢幕純裝飾。路由與 shell 關係以 `App.tsx` 為準。
