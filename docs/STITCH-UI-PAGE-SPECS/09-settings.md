# 系統設定

## 1. 頁面用途
- **做什麼**：系統層參數、帳號/安全相關、整合設定等（依實際 `settings.tsx` 分段）。
- **給誰**：`super_admin`、`marketing_manager`（頂欄「設定」可進）。
- **情境**：調整營運參數、檢視整合狀態。

## 2. 進入方式
- **route**：`/settings`。
- **從哪進**：頂欄 `Link`「設定」。
- **權限**：非 `cs_agent`。

## 3. 首屏必須看到的內容
1. **頁面標題**「系統設定」或同等。
2. **分段/Tab/Card**：至少一塊可操作的設定區（**後半細節待確認** `settings.tsx`）。
3. **儲存或套用**入口（若有表單區）。
4. **唯讀資訊**（版本、環境等— **待確認**）。
5. **錯誤/成功回饋**區或 toast 觸發點。

## 4. 區塊排列順序
- 建議：概要置頂 → 分類設定區塊由上而下或可 Tab 切換— **以實作為準**。

## 5. 按鈕盤點
### A. 主按鈕
- **儲存**、**套用變更**— **待確認**各區塊。
### B. 次按鈕
- 重新載入設定、重設預設— **待確認**。
### C. 危險
- 清除快取、撤銷整合— **待確認**是否 destructive + 確認 Dialog。
### D. 輔助
- 說明 tooltip、外部文件連結— **待確認**。

## 6. 篩選 / 控制
- 多為表單控制項：switch、select、input— **待確認**欄位清單。
- 無通用「列表篩選」除非有稽核 log 子頁— **待確認**。

## 7. 狀態 / badge / warning
- 未儲存變更提示、驗證錯誤 inline。
- 整合連線狀態 badge— **待確認**。

## 8. 可收合但不能消失
- 各設定區塊可 Accordion，但**類別標題與主欄位**不可無標籤移除。

## 9. 最不能被 AI 誤改
- 儲存語意與驗證（避免「自動儲存」未說明造成誤解— **若實作為手動儲存則保留按鈕**）。
- 破壞性操作須可辨識。

## 10. Stitch 用摘要
設定頁為主管/管理員專用，結構以分段表單或 Tab 為主，首屏須有頁名與至少一塊可編輯設定及儲存路徑。主 CTA 為儲存類；危險操作用色與確認不可拿掉。視覺可卡片化與分欄，但不可把設定藏進僅圖示選單、不可移除錯誤提示區塊。細節欄位請對照 `settings.tsx` 補齊盤點。
