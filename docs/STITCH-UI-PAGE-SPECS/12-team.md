# 團隊管理

## 1. 頁面用途
- **做什麼**：管理成員、角色、邀請或權限相關設定（依實際 `team.tsx`）。
- **給誰**：`super_admin`、`marketing_manager`（**待確認**是否開放 cs_agent 唯讀）。
- **情境**：新增成員、調整角色、停用帳號。

## 2. 進入方式
- **route**：`/team`。
- **從哪進**：側欄「團隊」或「團隊管理」。
- **權限**：依 `ROUTE_ACCESS`。

## 3. 首屏必須看到的內容
1. **頁面標題**。
2. **成員表**：姓名、email、角色、狀態— **待確認**欄位。
3. **邀請/新增成員**主按鈕。
4. **搜尋或篩選角色**— **待確認**。
5. **操作欄**：編輯、重設— **待確認**。

## 4. 區塊排列順序
- 頂工具列 → 表格 →（若有）分頁。

## 5. 按鈕盤點
### A. 主按鈕
- **邀請成員**、**儲存變更**（編輯 Dialog）。
### B. 次按鈕
- 重新整理列表— **待確認**。
### C. 危險
- **移除成員**、**停用**— destructive + 確認。
### D. 輔助
- 複製邀請連結— **待確認**。

## 6. 篩選 / 控制
- 角色 filter、狀態 filter— **待確認**。

## 7. 狀態 / badge / warning
- 邀請中、已啟用、已停用色票。

## 8. 可收合但不能消失
- 成員表不可改成僅頭像無角色資訊。

## 9. 最不能被 AI 誤改
- 角色變更與移除的確認流程。
- super_admin 人數或自我刪除防護— **待確認**後端規則於 UI 的反映。

## 10. Stitch 用摘要
團隊頁以成員表為核心，首屏含標題、篩選、表格與邀請/新增主 CTA。危險操作須可辨識並附確認。視覺可優化表格密度與行內操作改 dropdown，但不可移除角色欄、不可把移除藏成一般連結。細節以 `team.tsx` 為準。
