# 知識庫

## 1. 頁面用途
- **做什麼**：維護客服知識條目（FAQ、政策、商品說明等），供 AI/人工回覆引用。
- **給誰**：`super_admin`、`marketing_manager`、`cs_agent`（依 `ROUTE_ACCESS`）。
- **情境**：新增條目、修訂內容、分類整理、搜尋。

## 2. 進入方式
- **route**：`/knowledge`。
- **從哪進**：側欄「知識庫」。
- **權限**：全員可進；編輯權若有細分 **待確認**。

## 3. 首屏必須看到的內容
1. **頁面標題**與簡短說明。
2. **搜尋或篩選**：關鍵字、分類、品牌— **待確認**。
3. **條目列表**：標題、更新時間、狀態。
4. **新增條目**主入口。
5. **詳情/編輯區**（抽屜、右欄或導向子頁— **待確認**）。

## 4. 區塊排列順序
- 工具列 → 列表 → 詳情（或路由切換編輯頁）。

## 5. 按鈕盤點
### A. 主按鈕
- **新增**、**儲存**（編輯器）。
### B. 次按鈕
- 預覽、複製、匯入— **待確認**。
### C. 危險
- **刪除條目**— 應有確認。
### D. 輔助
- 格式化工具列（若 rich text）— **待確認**。

## 6. 篩選 / 控制
- 分類樹或 tag— **待確認**。
- 品牌 scope 與側欄關係— **待確認**。

## 7. 狀態 / badge / warning
- 草稿/已發布、同步失敗— **待確認**。

## 8. 可收合但不能消失
- 搜尋與新增入口不可收進無標籤選單。

## 9. 最不能被 AI 誤改
- 編輯器與儲存流程。
- 刪除確認。

## 10. Stitch 用摘要
知識庫頁首屏需有標題、搜尋/篩選、條目列表與新增主按鈕。詳情與編輯可側權或獨立頁，但儲存與刪除語意須清楚。視覺可改卡片式列表與閱讀寬度，但不可移除搜尋、不可把新增藏過深、刪除需確認。細節以 `knowledge` 相關頁面實作為準。
