# App Shell（側欄 + 頂欄）

## 1. 頁面用途
- **做什麼**：登入後全域框架——導航、品牌 scope、通知入口、設定／登出、角色相關戰情摘要。
- **給誰**：所有已登入使用者；側欄選項依 `role` 過濾。
- **情境**：任何主內容路由皆包於此 shell 內（除 login）。

## 2. 進入方式
- **route**：非單一路由；`AuthenticatedApp` 包裹。
- **從哪進**：登入成功後自動進入。
- **權限**：與各子路由一致；頂欄「設定」僅 `super_admin` | `marketing_manager`。

## 3. 首屏必須看到的內容
1. **側欄**：目前登入者頭像與名稱、角色標籤（管理員／主管／客服）。
2. **客服**：「我的工作」四格或擴充格（我的／待回／緊急／追蹤等）與「我的快捷」列表。
3. **主管**：「今日戰情」格狀、「團隊狀況」列表、「主管快捷」。
4. **品牌工作區**：`button-brand-selector`、渠道列表列點與 LINE/FB 色點。
5. **頂欄**：品牌/系統標題、`text-header-title`、通知（連 `/`）、設定連結（有權限時）、`button-logout`。

## 4. 區塊排列順序
- **左**：固定寬側欄 `data-testid="sidebar"`，上→下：登入者區 →（角色區塊）→ 品牌選擇器+渠道 → `nav` 功能選單 → 底欄渠道啟用狀態。
- **右**：頂欄 full width border-b → `main` 可捲動內容。

## 5. 按鈕盤點
### A. 主按鈕
- **側欄無全域單一 primary**；與頁面內 CTA 分離。導航以整列 `Link` 為主。
### B. 次按鈕
- 品牌下拉內「全部」`button-brand-all`、各 `button-brand-{id}`。
- 快捷視圖按鈕（切 `viewMode` 並 `setLocation("/")`）。
### C. 危險
- 側欄內無刪除鈕；**登出**在頂欄為重要操作但非 destructive 色（ghost）。
### D. 輔助
- 頂欄通知、設定連結；側欄選單項上未讀紅點（即時客服）。

## 6. 篩選 / 控制
- **品牌 scope**：`selectedBrandId` null = 全部；影響多頁 API。
- **viewMode**：僅影響 `/` 列表，透過快捷鈕設定。

## 7. 狀態 / badge / warning
- 在線綠點、待回/緊急數字顏色（紅/琥珀等）。
- 底欄「N 個渠道啟用中」與灰點「尚未設定渠道」。
- **不可**弱化到與背景同色。

## 8. 可收合但不能消失
- 團隊狀況 `max-h-[140px] overflow-y-auto` 已為收合式捲動；區塊本身不可刪。
- 功能選單可視覺精簡但**每個可見 role 的入口**必須保留。

## 9. 最不能被 AI 誤改
- 側欄寬度與對比（深色底）——與主內容區隔。
- 品牌選擇器與「全部」選項——刪除會誤判資料 scope。
- 登出必須可達，不可收進三層選單。

## 10. Stitch 用摘要
App Shell = 左 260px 深側欄 + 右頂欄 + main。側欄含使用者、依角色顯示戰情格與快捷（導向 `/` 篩選）、品牌下拉與渠道狀態、功能選單 Link。頂欄含標題、通知、設定（主管/管理員）、登出。無單一全站 primary 按鈕；主互動為導航列點擊與品牌切換。不可刪品牌「全部」、不可把登出藏沒、不可把側欄改成與主內容無對比。視覺可優化圖示與間距，但資訊架構與 role 差異需保留。
