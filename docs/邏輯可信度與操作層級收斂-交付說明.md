# 邏輯可信度與操作層級收斂 - 交付說明

## 一、真正造成數字不連動的原因

- **左側戰情摘要**（今日新進、待分配、緊急、逾時、結案、VIP待處理）與**員工摘要**（我的、待回、緊急、追蹤、逾時、結案）的數字來自 API：`/api/manager-stats`、`/api/agent-stats/me`。
- 前端在**改狀態、指派、改派、移回待分配、加標籤、發訊息、SSE 新訊息/contacts_updated** 等操作後，只做了 `queryClient.invalidateQueries({ queryKey: ["/api/contacts"], ... })`，**沒有**一併 invalidate `manager-stats` 與 `agent-stats/me`。
- 因此 React Query 仍使用舊的快取數字，直到下次 refetch（例如 15 秒間隔），導致「有勾選、切換、改狀態，但左側數字沒變」的**邏輯不可信**。

## 三、每個檔案改了什麼

### 1. `client/src/pages/chat.tsx`

- **數字連動**  
  - 新增 `invalidateContactsAndStats()`：同時 invalidate `/api/contacts`、`/api/manager-stats`、`/api/agent-stats/me`。  
  - 以下情境在成功後都改為呼叫 `invalidateContactsAndStats()`（必要時搭配 invalidate messages）：  
    - SSE `new_message`、`contacts_updated`  
    - 發送訊息、轉人工、恢復 AI、改狀態、移回待分配、指派、改派、釘選、客服標記、問題類型、加/移除標籤、發送評價、上傳傳圖  
  - 依賴陣列中補上 `invalidateContactsAndStats` 的 useCallback / useEffect。

- **篩選收斂**  
  - 上方主 tab 僅保留 5 個：**全部、我的案件、待我回覆、緊急案件、待追蹤**（移除「逾時未回」「待分配」tab；這兩項改由左側主管快捷進入）。

- **案件狀態 vs 系統標記**  
  - 狀態 Select 改為兩個 **SelectGroup**：  
    - **案件狀態**：待處理、處理中、等客戶回覆、已解決待觀察、已結案、已重開（`CASE_STATUS_VALUES`）。  
    - **系統標記**：待人工接手、新案件、已分配、待補資訊、待訂單編號、高風險（`SYSTEM_MARK_VALUES`）。  
  - 仍寫入同一個 `status` 欄位，僅在 UI 上分組，概念拆開。

- **視覺收斂**  
  - 列表卡：逾時改為橘色系（border/背景），不再用紅色；待分配改為琥珀。  
  - 每張卡重點 badge 收斂：第三行只突出「逾時」「緊急」「VIP」與追蹤/稍後（次層）；第一行移除重複的 VIP badge（VIP 改在第三行顯示）。

### 2. `client/src/components/app-sidebar.tsx`

- **主管戰情**  
  - 標題改為「今日戰情」；摘要卡文案：今日新進、待分配、**緊急**、**逾時**、**結案**、**VIP待處理**。  
  - **待分配 / 緊急 / 逾時** 三張摘要卡改為可點，點擊後 `handleViewShortcut` 切到對應 viewMode 並導向 `/`。

- **主管快捷**  
  - 只保留 **待分配、逾時未回、全部案件** 三個按鈕（移除「緊急案件」大按鈕；緊急改由摘要卡可點進入）。

- **員工摘要**  
  - 標題改為「我的工作」；文案縮短：我的案件→**我的**、待我回覆→**待回**、緊急案件→**緊急**、待追蹤→**追蹤**、逾時未回→**逾時**、今日結案→**結案**。

### 3. `shared/schema.ts`

- 新增 **案件狀態** 與 **系統標記** 常數：  
  - `CASE_STATUS_VALUES`：pending, processing, waiting_customer, resolved_observe, closed, reopened。  
  - `SYSTEM_MARK_VALUES`：awaiting_human, new_case, assigned, pending_info, pending_order_id, high_risk。  
- 註解標明：案件狀態為流程用、系統標記與案件狀態分開展示、後端仍為同一 status 欄位。

## 四、左側摘要數字現在怎麼計算

- **來源**：僅來自後端 API，不再用前端局部 state 或靜態值。  
  - 主管：`GET /api/manager-stats`（可帶 `brand_id`）→ today_new, unassigned, urgent, overdue, closed_today, vip_unhandled, team。  
  - 員工：`GET /api/agent-stats/me` → my_cases, pending_reply, urgent, overdue, tracking, closed_today。  
- **計算邏輯**：由後端依同一套規則（未結案、需人工、最後一則為 user、is_urgent、is_overdue、VIP 等）在 contacts 上統計，前端只顯示 API 回傳數字。

---

## 五、哪些操作會觸發摘要同步刷新

下列操作成功後都會呼叫 `invalidateContactsAndStats()`，因此會 refetch `/api/manager-stats` 與 `/api/agent-stats/me`，左側數字會更新：

1. 改案件狀態（狀態 Select）  
2. 指派 / 改派 / 移回待分配  
3. AI 或後端更新案件（經 SSE `contacts_updated` 或 new_message 後 invalidate）  
4. 標記／取消緊急（標籤或狀態變更）  
5. 問題類型更新  
6. 結案／重開（狀態變更）  
7. 新訊息進來（SSE new_message / 本機發送訊息）  
8. 轉人工、恢復 AI、釘選、客服標記（稍後/追蹤）、加/移除標籤、發送評價、上傳傳圖  

---

## 六、上方 tab 與左側快捷現在如何收斂

- **單一篩選狀態**：仍由 `ChatViewProvider` 的 `viewMode` 控制，上方 tab 與左側快捷共用同一套。  
- **上方主 tab（5 個）**：全部、我的案件、待我回覆、緊急案件、待追蹤。不再顯示「逾時未回」「待分配」。  
- **左側主管快捷（3 個按鈕）**：待分配、逾時未回、全部案件。  
- **左側主管摘要卡**：待分配、緊急、逾時 三張可點，點擊即切到對應 viewMode（等同 tab/快捷），緊急不再做一個大快捷按鈕。  
- 點 tab 或點左側快捷/摘要卡，都會更新同一個 viewMode，列表與高亮一致，不會兩套打架。

---

## 七、案件狀態與系統標記現在如何拆開

- **同一個 status 欄位**：後端仍只存一個 `status`，沒有新增欄位。  
- **UI 拆成兩組**：  
  - **案件狀態**：待處理、處理中、等客戶回覆、已解決待觀察、已結案、已重開（流程主狀態）。  
  - **系統標記**：待人工接手、新案件、已分配、待補資訊、待訂單編號、高風險（屬性/標記類）。  
- 同一個 Select 內用兩個 **SelectGroup** + **SelectLabel** 區分，選任一選項仍寫入 `PUT /api/contacts/:id/status` 的 `status`。

---

## 八、視覺上收斂了哪些地方

1. **紅色使用**：僅保留給真正緊急/逾時/高風險；列表卡「逾時」改橘色系，「待分配」改琥珀；「待我回覆」維持琥珀。  
2. **列表卡重點 badge**：每張卡最多突出 3 個重點（負責人/待分配、待我回覆/狀態、逾時/緊急/VIP）；VIP 不在第一行重複；優先/稍後/追蹤改為次層或合併顯示。  
3. **左側摘要文案**：主管「今日戰情」、緊急/逾時/結案/VIP待處理；員工「我的工作」、我的/待回/緊急/追蹤/逾時/結案，字數縮短、更戰情化。

---

## 九、我要怎麼驗收

1. **數字連動**  
   - 改一筆案件狀態 → 左側對應摘要數字應在短時間內更新。  
   - 指派 / 改派 / 移回待分配 → 待分配、我的案件等數字應更新。  
   - 發送一則訊息或由 SSE 收到新訊息 → 待回覆等相關數字應更新。  

2. **篩選收斂**  
   - 上方只看到 5 個 tab：全部、我的案件、待我回覆、緊急案件、待追蹤。  
   - 左側主管快捷只有 3 個：待分配、逾時未回、全部案件。  
   - 點左側「待分配/逾時/全部」或點摘要卡「待分配/緊急/逾時」→ 中間列表與上方 tab 高亮一致。  

3. **狀態與標記**  
   - 點開案件狀態 Select，應看到兩組：「案件狀態」與「系統標記」，選項分組清楚、不混在一起。  

4. **視覺**  
   - 列表卡不過多紅、重點 badge 不超過約 3 個、左側文案簡短。

---

## 十、驗收成功標準

- 改變案件狀態後，左側摘要數字**真的會變**。  
- 指派 / 改派 / 移回待分配後，左側摘要數字**真的會變**。  
- 新訊息進來時，該類摘要數字**會跟著變**。  
- 上方 tab 與左側快捷**不再重複打架**，共用同一 viewMode。  
- 狀態與標記**不再混在同一個不分組的 dropdown**，已拆成「案件狀態」與「系統標記」。  
- 畫面比改動前**更清楚、更穩**，紅色與 badge 收斂，左側文案更短、更戰情化。
