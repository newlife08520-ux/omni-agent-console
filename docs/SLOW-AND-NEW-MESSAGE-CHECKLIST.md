# 開很慢 ＋ 找不到新訊息 — 一頁排查

## 一、打開軟體好慢

### 先做這兩項（多數情況會明顯改善）

1. **重新整理一次**  
   關掉多餘分頁，只留一個中控台分頁，按 **F5** 或 **Ctrl+R** 完整重整。避免多個分頁同時輪詢、SSE 重連。

2. **左側選「全部」**  
   若目前選了某個品牌，改成 **全部** 再試。列表只會載入該品牌的聯絡人，資料量較小；「全部」雖然筆數多，但後端已有做優化（limit、索引），通常不會更慢。

### 若還是慢，請看環境

| 情境 | 可能原因 | 建議 |
|------|----------|------|
| **本機** `npm run dev` 很慢 |  port 被佔用、一頁商店同步在跑（約 30 秒後） | 關掉其他佔用 8080 的程式；開機後等約 1 分鐘再操作 |
| **Railway 正式機** 很慢 |  冷啟動、Volume I/O、Redis 連線 | 確認服務有跑 Node（非純靜態）；必要時升級 Volume、看 Deploy Log 是否有錯誤 |
| **只有「點開某則對話」時慢** |  已做：訊息 limit 80、AI 建議改背景、預取 on hover | 若仍慢，用 F12 → Network 看 `/api/contacts/:id` 與 `/api/contacts/:id/messages` 的耗時 |

## 二、找不到新訊息

### 1. 先看畫面上有沒有黃色橫幅

- 若出現 **「即時更新已中斷，新訊息可能不會自動出現」**：代表 SSE 斷線，**請按「重新整理頁面」**。  
- 重整後連線恢復，新訊息會再即時推過來；若沒重整，要等輪詢（約 5～10 秒）或手動重整才會看到。

### 2. 左側品牌選「全部」

- 新進訊息的聯絡人若**沒有對應到品牌**（例如 LINE 渠道 Bot ID 沒設對），只會出現在 **「全部」**。  
- 請把左上角品牌切到 **全部**，看是否出現該聯絡人／新對話。

### 3. LINE 有進訊息但中控台沒有

- 代表 **LINE Webhook 沒成功** 或 **沒打到你的伺服器**。  
- 請到 LINE Developers → 該 Channel → **Webhook URL** 設為：  
  `https://你的Railway網址/api/webhook/line`  
  並按 **Verify**，要顯示成功、不要 timeout。  
- 詳細步驟見：**TROUBLESHOOT-NEW-MESSAGE-NOT-SHOWING.md**、**RAILWAY-VERSION-AND-LINE-WEBHOOK.md**。

### 4. 快速檢查表

| 項目 | 怎麼確認 |
|------|----------|
| 即時連線 | 沒有黃色「即時更新已中斷」橫幅；若有就重整 |
| 品牌篩選 | 左側選「全部」看有沒有該聯絡人 |
| LINE Webhook | URL 正確、Verify 成功、無 timeout |
| 後端有收到 | Railway Deploy Log 有 `[LINE WEBHOOK START]`、`new_message` 等 |

---

## 三、總結

- **開很慢**：先重整、左側選「全部」；本機注意 port 與一頁商店同步；Railway 確認跑 Node、Volume 與 Log。  
- **找不到新訊息**：先看有沒有黃色斷線橫幅 → 重整；再選「全部」；最後確認 LINE Webhook URL 與 Verify 成功。
