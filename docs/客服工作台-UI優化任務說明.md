# 客服工作台 UI 優化 — 任務說明（下一輪直接丟給 Cursor 用）

## 設計原則（不追求「酷」，追求這 5 件事）

1. **一眼知道現在該先回誰**  
   首頁/列表要有很清楚的：待處理、未回覆、VIP、超時未回、已分配給我。不能所有對話長一樣。

2. **一眼知道這個客戶目前誰在接**  
   右側或列表裡，負責客服要非常清楚：姓名、頭像、在線/離線、是否上班、是否可轉派。不能再讓「待分配 / 已分配 / 小標籤」看起來混亂。

3. **客服不用思考就知道下一步按哪個**  
   指派、改派、移回待分配、標記待追蹤、已回覆/等客戶、結案，要放在很順手的位置，不要藏來藏去。

4. **大量案件時，畫面不能亂**  
   要有很清楚的：分區、狀態色彩、排序規則、篩選器、我的案件視角。這些比 fancy 動畫重要 100 倍。

5. **右側資訊要像「戰情面板」**  
   右邊不應只是基本客資，應包含：負責客服、指派方式（自動/手動）、問題分類、最後客戶發言時間、最後客服回覆時間、是否待追蹤、是否 VIP、是否高風險。讓客服和主管都好用。

---

## 建議優先做的 6 項優化

### 1. 左側列表改成更有層級感

每筆對話至少一眼看到：

- 客戶名  
- 平台  
- 最後一句摘要  
- 當前狀態  
- 是否待我處理  
- 負責客服  
- VIP / 高優先標籤  

### 2. 加「我的案件 / 全部案件 / 待處理 / 超時未回」快捷切換

這超重要，不然客服只會被訊息海淹沒。

### 3. 右側負責人員區塊做得更清楚

例如：

- 頭像  
- 姓名  
- 狀態（在線 / 離線）  
- 上班狀態  
- 指派方式（自動 / 手動）  
- 目前負載  

這樣改派與派單會超直覺。

###  4. 對話區底部操作列整理

現在底部按鈕如果太多，很容易亂。應把最常用的幾個做主操作，其他收進次級選單。

### 5. 顏色與標籤邏輯統一

例如：

- **紅色**：超時 / 高風險  
- **橘色**：待處理  
- **藍色**：已分配  
- **綠色**：已回覆 / 正常  
- **灰色**：待分配 / 離線  

不要每個地方自己發明一套顏色。

### 6. 改派 / 指派彈窗要更資訊化

每個客服選項要看到：

- 頭像  
- 名字  
- 在線/離線  
- 上班/下班  
- 當前處理量  

這樣才知道該派給誰。

### 顏色／標籤對照表（全站統一）

| 語意 | 顏色 | 用途 |
|------|------|------|
| **紅** (danger) | `bg-red-100 text-red-700 border-red-200` | 超時未回、高風險、逾時重分配 |
| **橘** (warning) | `bg-orange-100 text-orange-700 border-orange-200` | 待處理、待我回覆 |
| **藍** (assigned) | `bg-blue-100 text-blue-700 border-blue-200` | 已分配、等客戶回覆 |
| **綠** (normal) | `bg-green-100 text-green-700 border-green-200` | 已回覆／正常 |
| **灰** (muted) | `bg-stone-100 text-stone-600 border-stone-200` | 待分配、已結案、離線 |

### 驗收方式

1. **左側**：切換「全部／我的／待處理／超時未回」，列表內容與排序符合預期；每筆能一眼看出客戶名、平台、最後一句、狀態 pill、負責人、VIP/高優先。
2. **右側負責人員**：有負責人時顯示戰情卡（頭像、姓名、在線、上班、指派方式、負載）；無則顯示待分配與指派鈕。
3. **右側戰情摘要**：有最後發言、最後客服回覆、標記、VIP、高風險等欄位且正確。
4. **底部**：主列為 快捷回覆、附件、更多、輸入框、傳送；點「更多」可發送 AI/真人評價卡片。
5. **顏色**：左側狀態 pill 依語意使用紅/橘/藍/綠/灰，無自創色。

### 驗收成功標準

- 客服能一眼看出該先回誰（待處理／超時未回／我的）。
- 負責人區塊能一眼看出誰在接、在線與負載。
- 指派／改派彈窗能依頭像、在線、上班、處理量做決策。
- 大量案件時列表不亂、分區與色彩一致。
