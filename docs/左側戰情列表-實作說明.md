  - 超時未回時狀態顯示「超時未回」、並有「超時」標籤。

## 4. 我的案件 / 待我回覆 / 高風險 / 待分配 怎麼判斷

- **我的案件**：`assigned_agent_id === authUser.user.id`，且排除已結案（status 非 closed/resolved）。
- **待我回覆**：`last_message_sender_type === "user"` 且 status 非 closed/resolved（最後一則為客戶發言、尚未回覆）。
- **高風險**：`status === "high_risk"` 或 `case_priority != null && case_priority <= 2`，且未結案。
- **待分配**：`!assigned_agent_id && needs_human === 1`（需人工但尚未指派）。

---

## 5. 排序規則

依序比較，先符合者排前面：

1. **超時未回**：待我回覆且最後客戶發言時間 > 1 小時。
2. **高風險**：如上 isHighRisk。
3. **待我回覆**：needMyReply(c)。
4. **VIP**：vip_level 高者在前。
5. **最後訊息時間**：`last_message_at` 新→舊。

---

## 6. 怎麼驗收

1. **不點右側即可看出負責人**：左側每筆有「小頭像+客服姓名」，且與右側一致。
2. **不點右側即可看出優先程度**：每筆有「優先高/中/低」或無。
3. **一鍵「我的案件」**：點「我的案件」只顯示 assigned_agent_id = 當前使用者的案件。
4. **一鍵「待我回覆」**：點「待我回覆」只顯示最後一則為客戶發言的未結案案件。
5. **主管看高風險與待分配**：點「高風險」只顯示高風險未結案；點「待分配」只顯示未指派且需人工。
6. **大量案件仍清楚**：排序為 超時→高風險→待我回覆→VIP→時間；高風險有紅邊與「⚠ 高風險」；「我」與「超時」標籤明顯。

- 5 個快捷篩選（全部、我的案件、待我回覆、高風險、待分配）行為正確。
- 列表狀態為統一 5 種：待分配、待我回覆、等客戶回覆、處理中、已結案。
- 排序符合：超時未回 → 高風險 → 待我回覆 → VIP → 時間。

## 8. 若資訊仍不夠用，下一步建議

- **最後客服回覆時間**：若要在左側顯示「最後客服回覆距今多久」，需 API 或 contact 列表帶上 `last_human_reply_at`，再在卡片加一列或小字。
- **SLA 倒數**：若需「距 SLA 剩幾分鐘」，需列表帶 `response_sla_deadline_at` 並在前端算剩餘時間。
- **未讀/新訊息數**：若需「此對話有幾則未讀」，需後端提供未讀數或最後已讀訊息 id，再在卡片顯示數字。
