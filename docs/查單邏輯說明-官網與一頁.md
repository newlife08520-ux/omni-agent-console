# 查單邏輯說明：官網 vs 一頁式、為何官網查不到

## 一、目前「可以」查官網訂單嗎？

**可以。** 系統有接官網（SHOPLINE）查單，但需同時滿足：

1. **品牌有綁定官網**：該品牌在「設定 → 品牌與渠道」已填 **Shopline 商店網域** 與 **API Token**。
2. **客戶有說「官網」**：對話裡要出現「官網」「官方網站」「官網購買」等，系統才會用「只查官網」的模式（`preferShopline`）；否則會先查一頁、再查官網或合併。
3. **查單方式**：
   - **有訂單編號**：會用 `lookup_order_by_id`，優先走官網查該單號。
   - **只有手機**：若偵測到官網關鍵字，會用 `lookup_order_by_phone` 且 **只查官網**（不混一頁結果）。

---

## 二、為什麼官網會「查不到」？

可能原因與對應關係如下。

| 原因 | 說明 |
|------|------|
| **1. 品牌沒綁官網** | 該品牌未填 Shopline 商店網域 / API Token，或 Token 過期、錯誤，就不會打官網 API。 |
| **2. 客戶沒講「官網」** | 若只說「我要查訂單」+ 手機，系統會先查一頁商店；一頁有結果就回一頁，不會自動再查官網。要「只查官網」必須對話裡有官網關鍵字。 |
| **3. 官網 API 是「搜尋」不是整庫** | 官網手機查單用的是 SHOPLINE `/orders/search`，把「手機」當搜尋關鍵字，最多掃 30 頁（約 3000 筆）。若訂單很多、或訂單較舊不在搜尋前段，就可能搜不到。 |
| **4. 本地索引只有最近 N 天** | 官網訂單也會同步到本地（`sync-orders-normalized.ts`），預設 **90 天**，`--backfill` 才 365 天。若訂單超過同步範圍，本地沒有，就只能靠 API 搜尋；搜尋又受上面第 3 點限制。 |
| **5. 手機格式不一致** | 官網比對是「數字完全一致」。客戶留 09xx、系統正規化後若與 SHOPLINE 回傳格式不同，可能對不到。 |

**實務建議**：  
- 先確認該品牌已正確設定 Shopline 並可連線（後台有「測試官網」可按）。  
- 客戶要查官網時，請他**明確說「官網買的」或「官方網站」**。  
- 若訂單量很大或常查舊單，可考慮提高 sync 天數或跑 `--backfill`，並留意 SHOPLINE 搜尋 API 的筆數上限。

## 三、一頁式：可以「只輸入手機」就查嗎？還是要商品名＋手機？

**兩種都可以，依你提供的資訊選不同工具。**

| 情境 | 使用工具 | 規則摘要 |
|------|----------|----------|
| **只有手機、沒商品名** | `lookup_order_by_phone` | 可以只輸入手機就查。系統會掃一頁商店多個日期視窗（1/3/7/30/90/180 天）合併結果。若 **只命中 1 筆** 會直接回那一筆；若 **多筆** 會列摘要，請客戶補訂單編號或商品關鍵字，不會擅自幫他選一筆。 |
| **有商品名＋手機** | `lookup_order_by_product_and_phone` | 用商品關鍵字＋手機，先查本地索引（同品牌、同來源），可精準鎖定「某商品＋該手機」的訂單。適合一頁多商品、同手機多筆時縮小範圍。 |

也就是說：  
- **一頁式可以只輸入手機就查**，不需強制商品名。  
- 若同手機有多筆，系統會列摘要或請補「訂單編號／商品／要看哪一筆」，不會單筆定案（Phase 29/30 規則）。

## 四、目前查單邏輯總覽（誰先、誰後）

1. **有訂單編號**  
   - 一律用 `lookup_order_by_id`。  
   - 若客戶說過官網或我們推斷官網，會**優先查官網**該單號；沒有再查一頁。

2. **只有手機、沒單號**  
   - 用 `lookup_order_by_phone`。  
   - 若對話含「官網／官方網站」等 → **只查官網**（`preferShopline`），查無就回「官網查無」，不會改回一頁結果。  
   - 若沒說官網 → 先查一頁、再查官網，合併去重後回傳（或依品牌/設定）。

3. **有商品名＋手機**  
   - 用 `lookup_order_by_product_and_phone`（一頁／官網都會查，依來源過濾）。  
   - 本地索引有就先從本地篩「商品＋手機」，不足再打 API。

4. **有日期＋聯絡方式**  
   - 用 `lookup_order_by_date_and_contact`（依日期區間＋聯絡方式查）。

5. **「還有其他訂單嗎」／「全部訂單」**  
   - 若目前是單筆 context，會觸發 Phase 29 的「單筆展開多筆」：用同一支手機再跑一次 `lookup_order_by_phone`（或官網 only 時只查官網），結果多筆就變成多筆摘要。

6. **多筆時**  
   - 一律列摘要或逐筆，不自動選一筆；可說「最新那筆」「最早那筆」「第 N 筆」「全部訂單」或貼訂單編號指定。

## 五、快速對照：你想確認的事

- **官網現在能查嗎？**  
  → 能，但要：品牌有綁 Shopline + 客戶有講「官網」＋ 訂單在 API 搜尋或同步範圍內。

- **官網為什麼查不到？**  
  → 多數是：沒講官網（被當成一頁查）、品牌未綁/Token 問題、搜尋筆數/時間範圍限制、或手機格式不一致。

- **一頁式可以只輸入手機嗎？**  
  → 可以，只輸入手機就會用 `lookup_order_by_phone`；多筆時會列摘要不單筆定案。

- **查單邏輯是怎樣？**  
  → 如上第四點：有單號 → by_id；只有手機 → by_phone（可官網 only）；有商品+手機 → by_product_and_phone；其他還有日期+聯絡、更多訂單展開等。
