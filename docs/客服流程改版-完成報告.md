## 三、邏輯變更摘要

1. **先判斷再說話**：每則 user 訊息進 autoReplyWithAI 後，先 resolveConversationState → buildReplyPlan；再依 plan.mode 決定固定回覆或進 LLM。
2. **退換貨首輪**：primary_intent 為 refund_or_return / exchange_request / cancellation_request 且 isReturnFirstRound → mode = return_form_first，回傳固定「先承接＋表單連結＋可備註原因」，寫入 return_stage=1、waiting_for_customer=return_form_submit，不呼叫 LLM。
3. **F2**：return_form_first / return_stage_1 時 plan.must_not_include 含 F2 禁止句，並注入 system prompt「本輪硬規則 F2」；且 shouldNotLeadWithOrderLookup 時追加「不可先走訂單查詢為主流程」。
4. **24h 結案**：定時掃 last_message_at 且 last_message_sender_type=user、超過 idle_close_hours 的 contact；排除 closed/resolved/awaiting_human/high_risk；依情境發分流結語、寫 status=closed、close_reason=idle_24h、resolution_status=closed。
5. **評價**：僅在 PUT status=resolved|closed 且 isRatingEligible(contact) 時發送真人/AI 評價，並寫入 rating_invited_at。

## 五、驗收方式

1. **退貨首輪**：發「我想退貨」→ 應收到先承接＋表單連結，且無「官方/其他平台」、無先查單、無先轉人工。
2. **退貨＋單號**：發「我想退貨，MRQ55276」→ 仍先承接＋表單，不先走查單主流程。
3. **久候不想等**：發「太久了不要了」→ 先承接情緒＋導表單，不先懷疑通路。
4. **24h 結案**：某聯絡人最後一則為 user 且超過 24h 未回 → 定時 job 應發對應結語並設 status=closed、close_reason=idle_24h。
5. **已轉人工**：status=awaiting_human 之聯絡人 24h 未回 → 不應被 idle-close 關閉。
6. **評價**：將 status 改為 resolved/closed 時，僅在符合 isRatingEligible 時發送評價並寫入 rating_invited_at。

## 六、自我檢查與風險

- **idle_close_hours**：預設 24；若未寫入 settings，讀取為 null 時 getIdleCloseHours 回傳 24。
- **getContacts 與 last_message_sender_type**：idle-close 依 getContacts() 回傳之 last_message_sender_type 判斷「最後一則是否為 user」；若該欄位未正確填寫，可能誤關或漏關。
- **return_form_first**：依 primary_intent 與 return_stage 判斷；若 intent 誤判為非退換貨，則不會走固定表單流程。
