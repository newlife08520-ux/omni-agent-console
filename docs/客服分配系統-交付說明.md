# 客服分配系統 - 交付說明

## 一、已完成功能摘要

- **資料庫**：users 新增 `avatar_url`；contacts 新增 `assigned_at`、`assignment_method`、`needs_assignment`、`assignment_reason`、`response_sla_deadline_at`；assignment_history 新增 `action_type`、`operator_user_id`；設定檔預設 SLA、自動分配、逾時重分配開關。
- **分配邏輯**：僅分配給在線、可分配、工作時段內、未滿載且 `auto_assign_enabled` 的客服；策略為負載最少優先；支援手動指派、改派、移回待分配；逾時（SLA 分鐘）未回覆自動重分配。
- **午休/下班**：依全域時段判斷；無人可接時 AI 提示午休/下班並寫入系統訊息；時段由設定頁「人工客服服務時段」維護。
- **API**：`POST /api/contacts/:id/assign`（可帶 `agent_id` 手動指派）、`POST /api/contacts/:id/unassign`、`GET /api/contacts/:id/assignment`、`GET /api/team/available-agents`、`POST /api/team/:id/avatar`、`GET/PUT /api/settings/assignment-rules`。
- **背景**：每 60 秒執行逾時重分配掃描。
- **前端**：對話列表標籤（已分配/待分配/逾時重分配等）、聊天上方負責人與頭像、右側「客服處理資訊」、改派/移回待分配、改派選單顯示頭像與負載；團隊管理頁頭像顯示與上傳、在線/負載/自動分配標示；設定頁「人工客服服務時段」與「客服分配規則」（SLA、自動分配、逾時重分配）。

## 三、驗收要點（建議自測）

- **自動分配**：需人工案件有在線且可分配客服時，自動派給負載最少者；無人在線時保持待分配。
- **午休/下班**：午休或下班時段轉人工，無人可接時不派案，且 AI 系統訊息為午休/下班對應文案。
- **逾時重分配**：已分配案件超過 SLA 分鐘無人回覆，約 1 分鐘內被自動改派（若另有可分配客服）；assignment_history 有 reassign_timeout 紀錄。
- **手動操作**：主管可手動指派、改派、移回待分配；改派選單顯示客服頭像與「x/10 對話」。
- **顯示**：左側列表可見「已分配給 XXX」「待人工分配」等；聊天上方與右側可見負責人與頭像、指派方式、SLA 等；團隊頁可見頭像、在線、負載、自動分配。
- **頭像**：團隊頁點頭像可上傳 jpg/png/webp；即時客服改派選單與負責人區塊顯示頭像。
- **設定**：超級管理員可於設定頁調整「人工客服服務時段」與「客服分配規則」（SLA 分鐘、啟用自動分配、啟用逾時重分配）。

## 四、未完成／後續可補

- **品牌級客服時段**：目前僅全域時段；若需「每個品牌不同上下班/午休」，可新增 `business_hours_settings` 表與品牌維度 API。
- **客服個人上下班/午休**：目前以全域時段為主；若需每人不同，可再擴充 agent_status 或 users 的 work/lunch 欄位並在分配邏輯中使用。
- **在線心跳**：前端進入即時客服頁可呼叫 `PUT /api/agent-status/me` 設為在線；離開或定時心跳可依需求補上，避免長時間無操作被視為離線。
- **assignment_logs 專用表**：目前沿用 `assignment_history` 並加上 `action_type`、`operator_user_id`；若需完整獨立 assignment_logs 表可再遷移。

## 五、如何在本機測試分配（系統未上線時）

不需實際上線，用兩個瀏覽器（或一個一般 + 一個無痕）即可驗證「誰分配、自動分配、負責人員顯示」：

1. **準備兩個帳號**  
   - 帳號 A：主管或超級管理員（可點「分配」「改派」「移回待分配」）  
   - 帳號 B：客服人員（cs_agent）

2. **讓客服「在線、可接案」**  
   - 用帳號 B 登入 → 進入 **即時客服** 頁面（必要，才會更新在線狀態）  
   - 若有「客服狀態」或 **設定 → 客服狀態**，請設為 **在線、可分配**（或保持預設）  
   - 後端會依 `PUT /api/agent-status/me` 或進入頁面時更新 `is_online` / `last_active_at`

3. **觸發「需人工」並看是否自動分配**  
   - 用帳號 A 登入 → 即時客服 → 任選一則對話  
   - 點 **「轉人工」** 或 **「分配」**  
   - **有在線客服時**：應自動派給負載最少者，右側會出現 **負責人員**（頭像 + 姓名），左側該則顯示 **已分配｜XXX**  
   - **沒有在線客服時**：會顯示 **待分配**，右側負責人員區塊顯示「待分配」+ 按鈕 **指派**

4. **檢查負責人員是否像圖1**  
   - 右側 **客戶資訊** 分頁：上方應有 **負責人員** 區塊（大頭像 + 姓名 + 指派方式）  
   - 若已分配：可點鉛筆圖示 **變更** 打開改派視窗  
   - 左側對話列表：該則應有 **小頭像 + 已分配｜客服名**

5. **手動指派／改派**  
   - 主管在右側「負責人員」點 **變更**，或上方工具列點 **改派** → 選擇另一位客服 → 確認  
   - 右側應更新為新客服姓名與頭像  
   - 點 **移回待分配** 後，負責人員應變回「待分配」

按以上步驟即可在本機確認：**會自動分配**、**誰被分配**、**負責人員顯示是否與圖1一致**，無需實際上線。
