# 貼圖／置頂框框／左側不刷新 — 完整說明與處理

---

## 一、為什麼看不到貼圖、只看到「[貼圖訊息]」？

### 原因

- LINE 傳來的**貼圖、音訊、位置、檔案**等非文字訊息，後端目前**只存成一段說明文字**，例如：`[貼圖訊息]`、`[音訊訊息]`、`[位置訊息]`、`[檔案訊息]`。
- 資料庫沒有存貼圖的圖片網址或 sticker ID，所以**畫面上無法顯示真正的貼圖圖片**，只能顯示這段文字（或改版後會顯示成「圖示 + 說明」）。

### 目前已做的改善

- 聊天視窗裡若是 `[貼圖訊息]`、`[音訊訊息]` 等，會改為顯示**小圖示 + 說明文字**，不再只是純文字，較容易辨識。

### 若未來要顯示「真正的貼圖圖」

- 需要後端在收到 LINE Webhook 的貼圖事件時，把 **packageId、stickerId** 存進資料庫，並提供貼圖圖檔網址（或由前端依 LINE 規則組出網址）。
- 前端再依這些資料渲染圖片。這屬於功能擴充，需另做開發。

## 二、為什麼有兩個人「有特別框框、然後置頂」？

### 框框（左側色條）

左側聯絡人列表每一筆的**左側色條**代表**案件狀態／優先程度**，不是「兩個人」專屬，而是依規則套用：

| 顏色 | 代表 |
|------|------|
| **藍色** | 目前選中的聯絡人 |
| **橘色** | **逾時未回**（超過回覆時限） |
| **紅色** | **緊急案件**（高優先級或標記為緊急） |
| **琥珀色** | **待分配**（需人工但尚未指派客服） |
| **灰色** | 一般案件 |

所以那兩位會「有框框」，是因為他們被系統判為：逾時、緊急、或待分配其中一種（或選中時變藍色）。

### 置頂（排在最上面）

列表**排序規則**是（依序比）：

1. **緊急** 的排最前  
2. 再來是 **逾時未回**  
3. 再來是 **待分配**  
4. 再來依 **VIP 等級**（高的在前）  
5. 最後依 **最後訊息時間**（新的在前）

所以那兩位會在最上面，是因為他們是「緊急／逾時／待分配」或 VIP 較高，**不是**因為有特別勾選「置頂」。

### 星號（★）

- 聯絡人左側的 **★** 是「**釘選**」：點一下可把該對話釘選，再點一下取消。
- 釘選後星號會變成**實心**，方便你快速找到重要對話；**釘選不會改變上面那套排序**，排序仍以緊急／逾時／待分配／VIP／時間為準。

**總結**：  
「特別框框」= 左側色條，代表該案件的狀態（逾時／緊急／待分配等）。  
「置頂」= 被排序規則排到最上面（緊急／逾時／待分配／VIP／時間），不是手動置頂；星號才是手動釘選。

## 三、左邊畫面都沒動、沒刷新、看不出新的人進來？

### 可能原因與對應處理

#### 1. 你正在看「某一個品牌」，但新聯絡人沒有歸到該品牌（最常見）

- 若 LINE 渠道的 **Bot ID** 填成 **User ID（U 開頭）**，而不是 LINE Webhook 的 **destination（Channel ID）**，新進來的聯絡人會被存成 **沒有品牌**（`brand_id = null`）。
- 此時你在左側選的是「**預設品牌**」（或任一單一品牌），列表只會顯示**該品牌**的聯絡人，所以**新的人不會出現在這個列表**。
- 他們只會出現在左側篩選為「**全部**」時。

**處理方式：**

1. 到 **系統設定 → 品牌與渠道**，找到該 LINE 渠道的 **Bot ID**。
2. 在 LINE 傳一則測試訊息，到 **Railway 部署日誌** 找：`[WEBHOOK] destination: xxxxx`。
3. 把 **Bot ID** 改成上面那個 **xxxxx**（和 destination 完全一樣），儲存。
4. 之後新訊息進來的聯絡人就會歸到該品牌，在「預設品牌」下就看得見。

#### 2. 即時更新（SSE）有收到，但畫面沒更新

- 後端有送 **contacts_updated**，前端也有呼叫 **invalidateContactsAndStats** 重新拉聯絡人列表。
- 若你仍覺得「沒動」：可先**手動重新整理頁面**（F5 或 Ctrl+R）確認是否為快取或網路延遲；或檢查瀏覽器 Console 是否有錯誤導致 React 沒重繪。

#### 3. 聯絡人列表有設定「每 5 秒輪詢」＋ SSE

- 列表除了收到 SSE 會重拉，也有 **refetchInterval: 5000**，理論上約每 5 秒會再打一次 API。
- 若新聯絡人一直沒出現，優先檢查 **Bot ID ＝ destination**（同上）。

### 建議排查順序

1. 先切到左側「**全部**」看是否能看到新進來的聯絡人。  
2. 若「全部」看得到、但某品牌看不到 → 幾乎可確定是 **Bot ID 填錯**，請依上面步驟改為 destination。  
3. 若「全部」也看不到 → 檢查 Railway 日誌是否有 `[WEBHOOK]`、`destination`，確認 Webhook 有打到後端且沒有 `NO MATCH for bot_id`。

## 四、總結對照表

| 現象 | 原因 | 處理方式 |
|------|------|----------|
| 只看到「[貼圖訊息]」、沒有貼圖圖 | 後端只存說明文字，未存貼圖圖 | 已改為圖示+說明；要真貼圖需擴充後端與前端 |
| 某兩人有特別框框 | 左側色條＝案件狀態（逾時/緊急/待分配） | 屬正常；可依顏色對照上表 |
| 某兩人排最上面 | 排序：緊急＞逾時＞待分配＞VIP＞時間 | 屬正常 |
| 星號 ★ | 手動釘選該對話 | 釘選/取消釘選用，不影響排序 |
| 左側沒刷新、看不到新人 | 多數為 Bot ID 填成 U 開頭，新人 brand_id=null | 將 Bot ID 改為日誌中的 destination；或先看「全部」 |

若你已改好 Bot ID 仍看不到新聯絡人，可提供：目前左側選的是「全部」還是「預設品牌」、以及 Railway 日誌中該則訊息的 `[WEBHOOK] destination:` 與 `NO MATCH` 有無出現，方便進一步對照。
