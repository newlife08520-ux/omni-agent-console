## 二、員工版介面重點

- **左側**：目前登入者卡（頭像、姓名、角色、在線、負載 X/Y）→ 我的工作摘要（我的案件、待我回覆、緊急案件、待追蹤、逾時未回、今日結案）→ 我的快捷（我的案件、待我回覆、緊急案件、待追蹤、待分配）→ 品牌工作區 → 功能選單。
- **中間列表**：每筆卡片顯示客戶名、平台、VIP、負責客服、是否我的、是否待我回覆、優先/緊急/超時、最後一句摘要與「誰說的」、時間、待分配。
- **篩選 tab**：全部、我的案件、待我回覆、緊急案件、待追蹤、待分配（逾時未回僅主管可見）。

---

## 三、主管版介面重點

- **左側**：身份卡 → 今日戰情摘要（今日新進、待分配、緊急案件、逾時未回、今日結案、VIP 未處理）→ 團隊狀況（每位客服在線/負載/待回數）→ 主管快捷（待分配、緊急案件、逾時未回、全部案件）→ 品牌工作區 → 功能選單。
- **右上角**：目前品牌、視角「主管」、緊急數/待分配數、通知、設定、登出。
- **列表排序**：緊急 → 逾時未回 → 待分配 → VIP → 最新訊息。

---

## 四、緊急案件判定邏輯（最小可行）

任一條件符合即標為緊急案件（`is_urgent` / 摘要「緊急案件」）：

1. 狀態為 `high_risk` 或 `case_priority` ≤ 2。
2. 案件標籤含：客訴、退款、轉主管、緊急案件。
3. VIP 且最後一則為客戶發言且逾時（> 1 小時未回）。
4. 超過 SLA（`response_sla_deadline_at` 已過）。

內部仍保留 `status`、`case_priority`、`tags` 等欄位；畫面上統一顯示「緊急案件」「⚠ 緊急」。

## 五、通知規則（規劃與顯示位置）

目前實作：右上角「通知」顯示未讀數量（`/api/notifications/unread-count`），點擊回到首頁。未實作完整即時推播。

**規劃—主管該收到通知的情況**  
新緊急案件、待分配堆積過多、逾時未回超過門檻、客戶要求轉主管、某客服負載過高、VIP 負向情緒、自動分配失敗。  
**規劃—客服該收到通知的情況**  
新案件分配給我、輪到我回、我的案件被標為緊急、主管改派、客戶再次追問、我的案件即將逾時。

之後可將上述條件接到現有通知 API / 推播，並在「通知」入口內列出項目。

---

## 六、工作摘要卡數量怎麼算

- **員工版**：呼叫 GET /api/agent-stats/me，後端依「目前登入客服」篩選聯絡人後計算：我的案件 = 未結案且 assigned_agent_id=我；待我回覆 = 最後一則為 user；緊急 = isUrgentContact；逾時未回 = isOverdueContact；待追蹤 = my_flag=tracking；今日結案 = closed_at 今日且 closed_by_agent_id=我。
- **主管版**：呼叫 GET /api/manager-stats（可選 brand_id），後端依全量或品牌篩選計算：今日新進、待分配、緊急、逾時未回、今日結案、VIP 未處理；team 陣列為各客服的 open_cases_count、pending_reply 等。

---

## 七、快捷標籤：新增 / 刪除 / 排序 / 保存

- **新增**：在案件上輸入標籤並新增時，會寫入案件並加入「常用快捷標籤庫」（localStorage `omni_agent_custom_tags`）。
- **刪除**：快捷列中「自訂」標籤旁有 ×，點擊即從快捷庫移除（removeCustomTag），重整後仍為移除後結果。
- **排序**：目前顯示順序為「系統預設標籤」+「自訂標籤」（自訂依 localStorage 陣列順序）；已提供 `setCustomTagsOrder`，之後可接上拖曳或上/下按鈕寫回 localStorage。
- **保存**：自訂標籤存於 localStorage，重整後仍存在。

---

## 八、品牌 / 工作區位置

品牌選取與通道清單放在**左側**（品牌工作區區塊），不在內容區中間；與員工/主管工作流一致，不會像插在內容區的全域控制。

---

## 九、右上角內容

- **員工版**：目前品牌/系統名、視角「客服」、通知（含未讀數）、設定（依權限）、登出。
- **主管版**：同上，並在通知前顯示「緊急 X | 待分配 Y」（輕量數字）。

---

## 十、驗收方式與成功標準

1. **員工版左側**：不再空，有登入卡、工作摘要（含真實數字）、快捷、品牌工作區、功能選單。
2. **主管版**：左側有今日戰情、團隊狀況、主管快捷；能一眼看出待分配、緊急案件、客服負載。
3. **用語**：全站不再僅顯示「高風險」，改為「緊急案件」等直覺用語。
4. **緊急案件**：任一符合條件（標籤/狀態/優先級/VIP 逾時/SLA 逾時）的案件在列表與摘要中顯示為緊急。
5. **工作摘要卡**：數字來自 API，不顯示「—」。
6. **快捷標籤**：可新增（加在案件時一併加入快捷）、可從快捷列移除自訂標籤、重整後仍存在。
7. **品牌/工作區**：在左側，不擋內容區。
8. **右上角**：有視角、通知、主管版有緊急/待分配數，不只時鐘。
9. **員工與主管**：重點不同（員工看我的/待回/緊急/追蹤，主管看戰情/團隊/待分配/逾時），整體風格一致。
