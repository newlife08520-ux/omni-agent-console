# 客服指派與值班機制 — 現況盤點

## 一、已存在功能

| 項目 | 現況 | 說明 |
|------|------|------|
| 新對話 assigned_to_user_id | ✅ 有 | 使用 `contacts.assigned_agent_id`，自動分配時寫入 |
| 指派時間 | ✅ 有 | `contacts.first_assigned_at` |
| 在線／上班／午休時段 | ⚠️ 部分 | `agent_status` 表有 on_duty、lunch_break、work_start_time、lunch_end_time 等，**無** is_online、last_active_at |
| 手動指派／改派 | ✅ 有 | POST `/api/contacts/:id/assign`、`/reassign`，前端改派按鈕與對話框 |
| 分配紀錄 | ✅ 有 | `assignment_history` 表，GET `/api/contacts/:id/assignment-history` |
| 午休／下班無人時提醒 | ✅ 有 | 全員不可接時寫入系統訊息「目前人工客服暫時忙碌中…」，**未區分**午休 vs 下班話術 |
| 對話列表顯示負責人 | ✅ 有 | 列表與標題列顯示 assigned_agent_name |

## 二、尚未具備

| 項目 | 說明 |
|------|------|
| 客服 is_online / is_available / last_active_at | 需補在 users 或 agent_status |
| 最大同時對話數 max_active_conversations | 分配時未檢查上限 |
| auto_assign_enabled | 未區分是否參與自動分配 |
| contacts.last_human_reply_at | 未更新，無法算 SLA |
| contacts.reassign_count | 未維護 |
| contacts.assignment_status | 目前僅用 status，無 unassigned/assigned/reassigned/waiting_human 欄位 |
| 逾時未回覆自動重分配 | 無 X 分鐘內無人回覆則重派邏輯 |
| 全域客服時段設定 | 無系統設定預設上班／午休／下班時間 |
| AI 午休 vs 下班不同提示 | 目前僅一句通用「暫時忙碌中」 |
| 前端：負載顯示 (3/10)、在線狀態、右側資訊卡標籤、列表標籤 | 部分有負責人，缺負載、在線、午休/下班標籤 |

## 三、補齊順序（依需求 P0 → P1）

- **P0**：補齊客服在線欄位 → 補齊 contacts 欄位與前端顯示 → 自動分配加入 is_online/is_available/負載上限/全域時段 → 午休／下班時段設定與 AI 不同提醒  
- **P1**：逾時未回覆重分配 → 手動改派負載與在線顯示 → 分配／重分配紀錄（已有，可補原因與顯示）
