# 訂單與對話雙向跳轉・最小可用版本 實作回報

## 1. 採用哪種綁定方式

- **contact_order_links 表**：`(contact_id, global_order_id, source)`，同一對只會有一筆（INSERT OR IGNORE）。
- **綁定時機**：客服在**該對話中**使用「訂單編號查詢 / 進階搜尋 / 商品+電話搜尋」且查有結果時，自動將當前選中聯絡人與查到的每筆訂單建立連結。
- 綁定後即可：
  - 在**對話頁**看到「查看訂單」、複製訂單連結/訂單編號（取該聯絡人已綁定的第一筆訂單）；
  - 在**訂單查詢結果**的每張訂單卡上，若有綁定聯絡人則顯示「查看對話」並可複製訂單編號/訂單連結。

---

## 2. 是否支援自動回寫備註

- **不支援**。  
- 盤點結果：目前一頁商店 / Shopline 僅有 GET 拉單，**沒有**呼叫平台 API 更新訂單備註。  
- 因此改為提供**一鍵複製「備註用格式」**，讓客服手動貼到外部訂單備註。

---

## 3. 若不支援，自動產生什麼可複製內容

- **備註用格式（一鍵複製）**  
  在對話頁標題列右側的「複製」下拉選單中，選「**複製備註用格式**」會複製：

  ```
  客服對話：CV-YYYYMMDD-XXXXX
  查看連結：https://<您的網域>/?contact=<聯絡人ID>
  ```

  - **對話代碼**：`CV-YYYYMMDD-XXXXX`（XXXXX 為 contact id 五位數補零，例如 `CV-20260304-00128`）。
  - **查看連結**：目前頁面 origin + pathname + `?contact=<contact_id>`，開連結即可直接進入該對話。

- 其他可單獨複製的項目（同一選單或訂單卡上）：
  - 複製對話連結  
  - 複製對話代碼  
  - 複製訂單連結（本系統「訂單查詢」分頁連結，帶 `?tab=orders&order=<訂單編號>`）  
  - 複製訂單編號

## 5. 怎麼驗收

### 5.1 對話 → 訂單

1. 登入後選一位聯絡人（對話頁）。
2. 右側切到「訂單查詢」，輸入一筆存在的訂單編號並查詢。
3. 確認：**同一對話下**查單成功後，該訂單與當前聯絡人已綁定（之後在「複製」選單會出現訂單連結/訂單編號）。
4. 點標題列「**查看訂單**」→ 應切到右側「訂單查詢」分頁。
5. 點標題列複製圖示（📋）→ 選「複製對話連結」/「複製對話代碼」/「複製訂單連結」/「複製訂單編號」/「複製備註用格式」→ 貼到記事本，確認格式正確（備註格式為兩行：`客服對話：CV-...`、`查看連結：https://...`）。

### 5.2 訂單 → 對話

1. 維持在「訂單查詢」分頁，且查詢結果中有**已綁定過**的訂單（即上一步在同一對話查過的那筆）。
2. 該訂單卡上應出現「**查看對話**」連結。
3. 點「查看對話」→ 應切回該聯絡人的對話、右側回到「客戶資訊」分頁，且網址為 `?contact=<id>`。
4. 訂單卡上的複製圖示：複製訂單編號、複製訂單連結 → 貼到記事本確認內容正確。

### 5.3 URL 與書籤

1. 複製「對話連結」（含 `?contact=<id>`），在新分頁開啟 → 應直接進入該聯絡人對話。
2. 複製「訂單連結」（含 `?tab=orders&order=<訂單編號>`），在新分頁開啟 → 應進入首頁且右側為訂單查詢、查詢框已預填該訂單編號（手動點「查詢」可帶出結果）。

### 5.4 綁定僅在「該對話查單」時建立

1. 先**不要**選聯絡人（或選 A），到訂單查詢用訂單編號查出一筆訂單（不切到任何對話再查）。
2. 該訂單卡上**不應**出現「查看對話」（因尚未綁定）。
3. 選定聯絡人 A，在同一頁訂單查詢再查同一筆訂單 → 查詢成功後，該訂單卡應出現「查看對話」，且點下去會到聯絡人 A 的對話。

---
