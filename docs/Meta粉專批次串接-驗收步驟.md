# Meta 粉專批次串接 — 人工驗收步驟

## 前置
- 以 **超級管理員** 登入。
- 備妥具 `pages_show_list` 權限的 Meta User Access Token（可從 [Graph API Explorer](https://developers.facebook.com/tools/explorer/) 取得）。

---

## 1. 取得粉專列表
1. 進入 **留言中心** → 點導航 **粉專批次串接**。
2. 在「User Access Token」欄位貼上 Token，點 **取得粉專列表**。
3. **通過**：下方出現「選擇粉專並指定品牌」區塊，且列出至少 2～3 個粉專（名稱可勾選）。

---

## 2. 批次匯入 2～3 個粉專
1. 勾選 **2～3 個**粉專。
2. 在「歸屬品牌」下拉選一個既有品牌。
3. 點 **一鍵建立（N 個粉專）**。
4. **通過**：畫面顯示成功筆數（或成功/失敗明細），無前端錯誤。

---

## 3. 驗證 channel 與 meta_page_settings 皆有建立
- **渠道**：進入 **設定 → 品牌與渠道**，選擇同一品牌，確認該品牌下出現剛匯入的粉專名稱（Messenger 渠道），數量與匯入數一致。
- **留言中心設定**：進入 **留言中心 → 粉專與 LINE 設定**，確認同一批粉專出現在列表中，且歸屬品牌正確。
- **通過**：上述兩處皆能看到新粉專，且筆數一致。

---

## 4. 驗證新匯入頁面預設：AI 關、自動留言關
- 在 **品牌與渠道** 點進任一個剛匯入的粉專（渠道卡／編輯），確認 **AI 回覆** 為關閉（或 is_ai_enabled = 0）。
- 在 **留言中心 → 粉專與 LINE 設定** 點進同一個粉專的設定，確認 **自動留言** 為關閉（auto_reply_enabled = 0 或介面顯示關）。
- **通過**：AI 關、自動留言關，符合「只收訊、不外發」預設。

---

## 完成
以上 1～4 皆通過即為本包驗收通過。
