# UI 範圍與非目標（Phase 4）

---

## 範圍（要做）

- 內部可讀、可管、可查的 **表單式** UI（Radix/shadcn 既有元件）。  
- 品牌級開關、情境分頁、工具多選、發布按鈕、歷史列表。  
- 對話頁 **除錯抽屜**（見 `DEBUG_VIEW_REQUIREMENTS.md`）。

---

## 非目標（不做）

- 拖拉畫布、節點連線、視覺化 workflow。  
- 多語系 Admin 完整 i18n（可先 zh-TW）。  
- 行動版完美適配（內部後台可用桌面優先）。  
- 即時協作編輯（cursors）。  
- 顧客端任何改動。

---

## 視覺

- 沿用現有 design token；**不**為本專案重做品牌視覺。

---
