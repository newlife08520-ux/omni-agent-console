# 留言中心客服工作台改版回報

## 1. 資訊層級如何重排

- **之前**：以「分流決策卡」為標題，先出現建議處理方式、負責人，品牌/粉專/商品多為小灰字或次要資訊，操作以「儲存」「標記已回覆」「轉客服」「隱藏」等較弱按鈕為主，LINE 導向資訊不明顯。
- **現在**：
  - **左側列表**：第一眼是「品牌｜商品｜留言類型」，再來是粉專／貼文、留言摘要、狀態｜導向結果；品牌與商品用較大、對比度高的字。
  - **右側**：一進來先看到「結論摘要」一句話與摘要區塊（品牌、粉專、貼文、商品、判定來源、留言類型、優先級、目前導向、狀態、建議動作），再來是「導向通道」顯性區塊，然後才是負責人、留言原文、回覆編輯與工作台操作按鈕。
  - **整體**：以「這則從哪來、問什麼、導哪條 LINE、處理到哪、是否敏感」為優先，表單與設定細節往後放。

---

## 2. 左側卡片改了哪些內容

- **第一行（最大最明顯）**：`品牌名稱｜商品名稱｜留言類型`  
  - 例：`KORENA｜粉鑽乳｜一般詢問`、`私藏生活｜皇的枕｜客訴`  
  - 品牌、商品不再小灰字；商品取 `detected_product_name` 或 `post_display_name`，無則顯示「未判定商品」；留言類型由意圖/優先級推導（客訴、敏感、一般詢問等）。
- **第二行**：粉專名稱、貼文名稱／標題（`page_name`、`post_name` 或 `post_display_name`）。
- **第三行**：留言摘要（`message`，最多兩行 truncate）。
- **第四行**：`狀態｜導向結果`  
  - 例：`未處理｜導 KORENA 一般 LINE`、`已隱藏｜導 私藏生活售後 LINE`、`待人工｜未判定商品`。  
  - 狀態由 `commentStatusLabel()` 統一（未處理、待送出、已自動回覆、已人工回覆、已隱藏、待人工、執行失敗）；導向由 `routingSummary()` 顯示目前導向哪條 LINE 或「未判定商品」等。

---

## 3. 右側摘要面板長什麼樣

- **位置**：右側詳情區最上方，先於任何表單。
- **第一行（一句話總結）**：  
  `品牌｜商品｜留言類型｜導向結果｜尚未送出/已送出`  
  例：`KORENA｜粉鑽乳｜一般詢問｜導 KORENA 一般 LINE｜尚未送出`
- **下方摘要區**（網格）：  
  品牌、粉專、貼文、商品、判定來源、留言類型、優先級、目前導向（哪個 LINE/商品頁/活動頁）、目前狀態、建議動作。  
  一眼可看到「從哪來、問什麼、導哪裡、現在狀態、建議下一步」。

---

## 4. LINE 導向如何顯示與切換

- **顯示**：  
  - 右側有獨立「導向通道」區塊（顯眼背景），直接寫：  
    - **目前導向**：一般 LINE / 售後 LINE / 商品頁 / LINE（未指定連結）/ 未判定／人工。  
    - **導向來源**：貼文 mapping、敏感規則、粉專預設等（由 `reply_link_source`、`classifier_source` 等推導）。
  - 若有 `target_line_value`，提供按鈕「複製／開啟導流連結」；詳情區工作台另有「立即導 LINE」可複製或開啟該連結。
- **切換**：  
  - 目前「快速切換」為：複製/開啟現有導流連結。  
  - 導向目標的設定（哪個粉專導一般 LINE / 售後 LINE）集中在「粉專與 LINE 導向設定」頁；文案說明：「快速切換需由粉專與 LINE 導向設定頁調整預設；此則可手動改回覆內容後送出。」  
  - 若未來後端支援單則留言改寫 `target_line_type` / `target_line_value`，可再於此區塊加一般 LINE / 售後 LINE / 商品頁 / 活動頁 / 人工 的快速切換按鈕。

---

## 5. 粉專與 LINE 設定頁如何設計

- **入口**：  
  - 留言互動中心內新增分頁「粉專與 LINE 導向設定」。  
  - 側欄新增「粉專與 LINE 設定」，連結至 `/comment-center#page-settings`，進入後直接開啟該分頁。
- **內容**：  
  - 單一列表（表格），一列一粉專，欄位：  
    - 品牌、粉專、一般 LINE、售後 LINE、敏感件預設、自動回覆、自動隱藏、預設商品、預設處理方式。  
  - 資料來源：`GET /api/meta-page-settings`（支援 `brand_id`），後端已回傳 `brand_name`。  
  - 一般 LINE / 售後 LINE 有值時顯示「已設定」（title 可帶連結），否則「—」；敏感件預設為「自動隱藏」或「不自動隱藏」；自動回覆／自動隱藏為「開」/「關」；預設處理方式為導商品頁／導 LINE／導活動頁等。
- **範圍**：本版僅做「列表一眼看出哪個粉專導哪個 LINE」，不在此頁做編輯表單（避免發散）；編輯仍可依現有後端 API 另做或之後補上。

---

## 6. 如何驗收這版是否真的更清楚

- **左側**：  
  - 掃一眼列表，是否能在 2 秒內回答：這則從哪個品牌/粉專來、在問哪個商品、是客訴還是一般詢問、目前狀態與導向哪條 LINE。  
  - 檢查品牌、商品是否明顯（不再小灰字）。
- **右側**：  
  - 點開一則留言，是否先看到「一句話總結」與摘要區，再看到導向通道與表單。  
  - 是否不用找就能看到「目前導向哪個 LINE」與「導向來源」；有連結時能否一鍵複製/開啟。
- **粉專與 LINE**：  
  - 從側欄點「粉專與 LINE 設定」或從留言中心切到該分頁，是否一眼看到「哪個粉專對應哪個一般 LINE / 售後 LINE」及開關、預設處理方式。
- **操作**：  
  - 是否具備「立即公開回覆」「立即隱藏留言」「立即導 LINE」「標記待人工」「標記已完成」等工作台式按鈕，且「立即隱藏」會真的呼叫 Meta 隱藏 API。

- **主狀態**（由 `commentStatusLabel()` 推導）：未處理、待送出、已自動回覆、已人工回覆、已隱藏、待人工、執行失敗。  
  （篩選下拉仍用後端既有：全部、未處理、已回覆、待人工、已隱藏、緊急／客訴。）
- **附加語意**：留言類型顯示為「客訴」「敏感」「一般詢問」等（由意圖與優先級推導）；導向結果顯示「未判定商品」「導 OOO 一般/售後 LINE」等。
