# 01 — 架構總覽與端到端資料流

## 系統角色

本專案（Omni-Agent-Console）是 **多品牌電商客服後台 + LINE／Messenger Webhook**。客人傳訊後，可能觸發 **AI 自動回覆**；查訂單時 AI 會呼叫 **Function Calling 工具**，工具向 **SuperLanding（一頁商店）** 或 **SHOPLINE Open API** 取單，再把結果格式化給客人。

## 端到端路徑（簡化）

1. **Webhook**（如 `line-webhook.controller.ts`，未收入此包）收到使用者訊息 → 建立／更新 `Contact`、寫入 `messages`。
2. **AI 回覆管線**（`ai-reply.service.ts`）：組 prompt、呼叫 LLM、處理 tool calls 迴圈。
3. **工具執行**（`tool-executor.service.ts`）：`lookup_order_by_id`、`lookup_order_by_phone`、`lookup_order_by_product_and_phone`、`lookup_order_by_date_and_contact` 等。
4. **統一查詢**（`order-service.ts`）：依品牌設定合併 SuperLanding／Shopline、本地索引快取。
5. **付款語意**（`order-payment-utils.ts`）：`derivePaymentStatus` → `success` / `failed` / `pending` / `cod` / `unknown`。
6. **對客一頁格式**（`order-reply-utils.ts`）：`formatOrderOnePage` 產生固定欄位文字（含姓名／電話／地址隱碼）。
7. **Deterministic 短路**：工具回傳帶 `deterministic_skip_llm` + `deterministic_customer_reply` 時，第二輪 LLM 可略過，直接對客顯示程式產生的全文，避免模型改寫或縮成兩行。

## 與「回覆查訂單」直接相關的檔案（此包內 `source/`）

- 編排與工具：`tool-executor.service.ts`、`ai-reply.service.ts`、`openai-tools.ts`

- 契約與旗標：`deterministic-order-contract.ts`、`order-feature-flags.ts`、`order-single-renderer.ts`、`order-multi-renderer.ts`
- 上下文：`order-active-context.ts`、`order-fast-path.ts`、`order-multi-selector.ts`、`order-product-filter.ts`
- 安全與清洗：`content-guard.ts`、`tool-llm-sanitize.ts`、`intent-and-order.ts`
- 型別：`schema.ts`
- 人格／規則摘要：`PHASE97_MASTER_SLIM.txt`

## 關鍵設計目標

- **同一套對客卡片**：工具給 LLM 的 `one_page_summary` 應逐字使用（prompt 與 `SYS_NOTE` 強調）。
- **雙資料源**：一頁商店與官網 Shopline 欄位結構不同，需在對應 mapper 與付款邏輯分開處理。
- **隱私**：對客訊息中姓名、電話、地址需隱碼（見 `tool-llm-sanitize` 與 `formatOrderOnePage`）。
