# 設計語言與不可違反規則

## 品牌與氛圍

- **後台營運工具**，不是消費型行銷站；優先 **可讀性、可掃描、狀態可辨識**，其次才是裝飾。
- 整體 **偏暖淺灰工作區**，深色側欄與淺色主內容對比清楚。

## 色彩（不可擅自改成冷色科技霓虹主題）

| 用途 | 方向 | 說明 |
|------|------|------|
| 主內容背景 | `#faf9f5` 同系 | 大面積底，減少刺眼白 |
| 卡片／列表底 | `white` / `stone-50` | 與背景區隔 |
| 邊框 | `stone-200` 為主 | 低對比分隔 |
| 側欄 | `stone-800` 底、白字 | 固定寬度約 **260px**（側欄元件） |
| 主行為（傳送等） | `emerald-600`／`emerald-700` hover | 與「成功／進行」語意一致 |
| 警示條（SSE 斷線等） | `amber-500` 底 | 可見但不等同錯誤紅 |
| 危險／緊急 | `red-*` | 緊急案件、錯誤、逾時 |
| 待處理／待回 | `amber`、`orange` | 與程式 `STATUS_SEMANTIC` 一致 |

**不可**：把側欄改成淺色而導致與主內容無邊界；把緊急狀態改成與「一般」相同色階。

## 字級與階層

- 頁內標題：`text-lg`～`text-sm font-semibold`
- 列表次要資訊：`text-[10px]`、`text-[11px]`、`text-xs` 常見
- **高密度列表**為常態：不可全面放大字級導致一屏可見案件數腰斬（可微調，需維持資訊密度等級）

## 間距與圓角

- 列表列、小卡：`rounded-xl` 常見
- 按鈕：`rounded-md`（shadcn Button 預設）
- 內距：列表 `p-2`～`p-3`，頁面邊 `p-4`～`p-5` 級

## 陰影與邊框

- 以 **細邊框**（`border-stone-200`）為主，重陰影少用
- Popover／dropdown 可用 `shadow-lg` 級

## 元件層級（全站）

1. **Primary**：一個視野內主要行為（例如「傳送」）用實心 emerald 或 `Button` default。
2. **Secondary**：outline／ghost，並列於工具列。
3. **Destructive**：刪除、不可逆 — `destructive` variant，需搭配確認。

## 全站共用（shadcn / new-york）

- 元件來自 `@/components/ui/*`（Radix + cva）
- **Token 以 CSS variables 為準**（`client/src/index.css` 的 `:root`）：`--primary`、`--destructive`、`--radius` 等；Tailwind 對應 `bg-primary`、`text-muted-foreground`。

## 圖示

- **lucide-react**，線條風格；側欄與按鈕尺寸多 `w-4 h-4`～`w-5 h-5`。

## 不可違反規則（Stitch 必守）

1. **不得刪除**即時客服的 **三欄語意**（列表｜對話｜右側脈絡／訂單），可改寬窄與收合，不可變單欄失能流程。
2. **不得隱藏**角色相關入口：客服必須能到「即時客服」；主管／管理員必須能到「設定／知識庫」等（依 `ROUTES.md`）。
3. **不得移除**關鍵狀態表達：待回、緊急、待分配、渠道（LINE／FB）、SSE／輪詢提示（可改文案長度，不可無提示）。
4. **對話輸入區**必須常態可見（選中聯絡人時）：輸入框、`data-testid="button-send-message"` 所代表之傳送行為需保留。
5. **知識庫頁**四大分頁語意需保留：**Prompt／圖像素材／行銷規則／沙盒**（見 `support-workbench-ui.txt`），不可合併成無標題單頁長文。
6. **無障礙**：可點擊列、按鈕需維持足夠觸控／點擊區；對比需維持可讀（WCAG 建議往 AA 靠攏，至少不自願降低對比）。

## 高資訊密度頁

- **即時客服**、**留言中心**、**知識庫**：不可「極簡」到 KPI 與狀態徽章消失。
