# support-diagnostics — 客服績效（路由 `/performance`）

> **第二輪可選**：與首輪 14 檔並用；路徑 `stitch-context/extra/`。

## 用途

- **客服**：檢視個人今日案件量、首次回覆、結案率等。
- **主管／管理員**：**戰情板**（待處理、緊急、待分配、結案率）、圖表、客服負載、主管報表。

## 首屏必須看到

- 標題「客服績效」+ 副標（依角色不同）
- **客服**：2×4（響應式）**StatCard** 網格
- **主管**：Tabs — **戰情板** | **全部客服績效** | **主管報表**

## 主操作

- 切換 Tabs（主管）
- 閱讀 KPI 與圖表（無單一「送出」；資料由 API 載入）

## 次操作

- 圖表 Tooltip、列表捲動、各客服卡片展開（見原始碼）

## 危險操作

- 本頁以 **唯讀儀表** 為主；若有導向其他頁之連結，不可在改版中移除而不替代

## 不可刪

- `data-testid="performance-page"` 所在根容器語意
- **異常與警示** Card（有 alerts 時）— 紅色系區塊
- 空狀態文案：「尚無狀態數據」「尚無客服數據」— 與「載入錯誤」需區隔
- 主管三 Tab 名稱語意
