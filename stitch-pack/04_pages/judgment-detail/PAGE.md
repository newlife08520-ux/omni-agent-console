# judgment-detail — 即時客服「對話＋脈絡」

**對應路由**：`/`（與 judgment-home **同一頁**）  
**原始碼**：`client/src/pages/chat.tsx`（副本與 home 相同檔案，閱讀時聚焦 **中央訊息區** 與 **右側面板**）

## 1. 頁面用途

顯示單一聯絡人之 **訊息時間序**，並提供 **回覆輸入** 與 **訂單／聯絡人脈絡**（右側或同屏區塊）。

## 2. 主要使用者

與 judgment-home 相同；此區為 **高專注工作區**。

## 3. 首屏必須看到的區塊

- 訊息列表（可捲動）、輸入區（textarea／input）、**送出**。
- 右側：**訂單摘要**、付款／履行狀態、可操作之案件欄位（若實作於此檔）。

## 4. 主操作按鈕

- **送出訊息**
- **附加檔案／圖片**（若 UI 有）

## 5. 次操作按鈕

- 複製內容、插入範本、emoji、更多選單（⋯）。

## 6. 危險操作

- 結案、轉派、關閉 AI 等（若有）：需清楚標示與二次確認。

## 7. 重要但目前可能亂

- 右側資訊過長、訂單多筆時層級不清；可 **分 tab／摺疊**，不可刪欄位語意。

## 8. 可收合但不能消失

- 訂單詳情、除錯用 metadata（若有）、長訊息「展開全文」。

## 9. 頁面狀態

| 狀態 | 說明 |
|------|------|
| loading | 切換聯絡人、載入訊息 |
| empty | 新對話無訊息 |
| partial | 訊息載入中但輸入區已顯示 |
| success | 訊息與右側脈絡一致 |
| error | 傳送失敗、載入失敗 |

見 `states/*.json`。
