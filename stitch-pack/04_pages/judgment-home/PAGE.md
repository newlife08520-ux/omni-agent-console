# judgment-home — 即時客服「列表／首屏」

**對應路由**：`/`  
**原始碼**：`client/src/pages/chat.tsx`（本資料夾內為副本，僅供 Stitch 閱讀）

## 1. 頁面用途

在 **不離開首屏** 的前提下，讓客服瀏覽、篩選、排序對話案件，並進入單一對話。

## 2. 主要使用者

- 客服（cs_agent）：我的案件、待回、緊急、追蹤、待分配視角。
- 主管：今日戰情、待分配、緊急、團隊視角（與側欄連動）。

## 3. 首屏必須看到的區塊

- 頂部 **view tabs** 或等價視角切換（與 `useChatView` 同步）。
- **搜尋／篩選**（若實作在列表上方）。
- **虛擬化或捲動列表**：每列含頭像、名稱、渠道、最後訊息預覽、時間、狀態徽章。

## 4. 主操作按鈕

- **點選列表列** → 載入該聯絡人對話（主路徑）。
- **篩選／搜尋確認**（若有明確按鈕）。

## 5. 次操作按鈕

- 重新整理列表、清除篩選、切換品牌（若在列表區有快捷）。

## 6. 危險操作

- 批量動作（若有）：需與確認對話框並陳，不可視覺弱化到誤觸。

## 7. 重要但目前可能顯得雜訊多

- 多徽章並列、多數字統計；可 **分組／層級化**，不可刪語意。

## 8. 可收合但不能消失

- 進階篩選、欄位選擇、小計／匯總條。

## 9. 頁面狀態

| 狀態 | 說明 |
|------|------|
| loading | 拉 contacts、品牌切換後 |
| empty | 無符合篩選之對話 |
| partial | 列表有資料但右欄尚未選取 |
| success | 列表與預覽／右欄一致 |
| error | API 失敗、權限問題 |

見 `states/*.json`（去敏感範例）。

## 實作備註

`chat.tsx` 單檔含 **列表＋對話＋右欄**；Stitch 優化列表區時請 **同步對照** `judgment-detail/PAGE.md` 避免破版。
