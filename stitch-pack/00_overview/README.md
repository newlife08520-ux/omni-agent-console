# Stitch UI Context Pack — Omni-Agent Console

本資料夾僅供 **Stitch／設計工具** 理解現有 **前端 UI**，不包含後端商業邏輯、不含 secrets、不含 `node_modules`／build 產物。

## 1. 這是什麼產品

**Omni-Agent Console（AI 客服／全通路中控台）**：內部營運用於處理 **即時對話**、**留言收件匣與規則**、**品牌／渠道設定**、**知識庫與 AI 設定**、**績效與數據**。  
口語上可對應「審判官」情境的是 **即時客服主畫面**：客服需在多狀態、多欄位下快速判讀與回覆。

## 2. 主要使用者是誰

- **客服人員（cs_agent）**：以 `/` 即時客服為主，需看案件列表、對話、訂單／狀態徽章、待回與緊急指標。
- **行銷經理／超級管理員**：可進設定、知識庫、數據、團隊、留言中心等。

## 3. 目前要優化哪些頁（第一輪）

見 `TARGET-PAGES.md`。對應程式路由：

| Stitch 資料夾名稱 | 實際路由 | 說明 |
|-------------------|----------|------|
| `judgment-home` | `/` | 即時客服：列表／篩選／主管與員工視角 |
| `judgment-detail` | `/`（同頁右欄／對話區） | 與首頁同一 `chat.tsx`，右側對話＋脈絡面板 |
| `support-workbench` | `/knowledge` | AI 與知識庫設定 |
| `support-diagnostics` | `/performance` | 客服績效（KPI／診斷型資訊） |

## 4. 哪些資訊架構不能改

見 `MUST-KEEP.md` 與各頁 `PAGE.md`。原則：

- **側欄導航順序與角色可見性**（與 `ROUTE_ACCESS`／sidebar `roles` 一致）不可擅自刪頁或合併成「假簡化」。
- **即時客服三欄（或類似）資訊層級**：列表 → 對話 → 右側脈絡／訂單，不可改成單欄而遺失並行判讀。
- **案件狀態、渠道、待回／緊急語意**必須保留可見性（可改視覺，不可移除語意）。

## 5. 哪些按鈕不能刪

見 `MUST-KEEP.md`。包含：**登出**、**發送訊息**、**品牌切換**、**主管／員工戰情數字入口**、留言中心各 tab 對應之核心操作等。

## 6. Debug／evidence／diagnostics 不可完全隱藏

- **內測模擬**（`/comment-center/simulate`）、**績效／數據**中的指標與篩選，屬營運與除錯情境；視覺可收合，但 **不可預設永久隱藏**。
- 對話區 **系統標記／案件欄位**（若畫面上有）為判讀依據，不可為「簡潔」整塊刪除。

## 7. 視覺可優化 vs workflow 不可變

- **可優化**：色票、字級階層、spacing、卡片圓角、表格密度、empty state 插圖、loading skeleton、圖示一致性。
- **不可變**：登入 → 選品牌（若有）→ 進主工作頁；從列表進對話再回列表；權限不足時不可顯示管理頁主功能。
