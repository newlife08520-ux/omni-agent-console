# 第一輪最重要頁面（給 Stitch）

## 1. judgment-home（`/` 列表與篩選區）

- **目標**：讓客服 **10 秒內** 找到下一通該處理的對話。
- **首屏最重要**：品牌選擇、view tab／快捷、搜尋／篩選、案件列表列（狀態、渠道、最後訊息、時間）。
- **核心 CTA**：點選一列進入對話（非單一 named button，是 **整列可點** 行為）。
- **不可誤刪**：待回／緊急／我的案件 等 **視角切換**、**渠道與狀態徽章**、**SLA 或逾時提示**（若有）。

## 2. judgment-detail（`/` 對話＋右側脈絡）

- **目標**：在 **不離開本頁** 下完成回覆與查單脈絡判讀。
- **首屏最重要**：訊息時間序、輸入框與送出、右側（或同屏）訂單／聯絡人摘要。
- **核心 CTA**：**送出訊息**、**附加**（若有）、切換 **AI／人工** 或類似模式（若畫面有）。
- **不可誤刪**：**訊息氣泡與時間**、**右側訂單卡片**、**付款／履行狀態標籤**、**案件操作**（指派、狀態、備註等 — 以實際 UI 為準）。

## 3. support-workbench（`/knowledge`）

- **目標**：可調整 AI／知識而不誤觸 production 行為說明清楚（視覺上區隔「高風險設定」）。
- **首屏最重要**：分區標題、主要表單區、儲存／預覽相關按鈕。
- **核心 CTA**：儲存、預覽 prompt（若有）。
- **不可誤刪**：**system／品牌 prompt 區塊**、**測試 id 或警告文案**（若有）。

## 4. support-diagnostics（`/performance`）

- **目標**：主管／客服能 **自我檢核** 產能與品質指標。
- **首屏最重要**：時間範圍、核心 KPI 卡片或表、個人 vs 團隊切換（若有）。
- **核心 CTA**：篩選日期、匯出（若有）。
- **不可誤刪**：**指標定義或 tooltip**、**空狀態說明**（避免以為沒資料＝沒問題）。
