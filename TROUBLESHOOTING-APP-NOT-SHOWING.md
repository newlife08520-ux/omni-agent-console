# 網頁打開但 APP 不出現 — 排查步驟

## 可能原因與對應檢查

### 1. 網址或連線對象不對
- **狀況**：目前 server 只聽 `127.0.0.1`，只有「跑 npm run dev 的那台電腦」能連。
- **請確認**：
  - 是用**同一台電腦**的瀏覽器開。
  - 網址是 **`http://localhost:5001`** 或 **`http://127.0.0.1:5001`**（port 要與終端顯示一致）。
- **若是用手機或另一台電腦開**：畫面會連不上或無法載入，需要改 server 綁定（見下方「可選修正」）。

### 2. 白畫面（完全空白）
通常代表：HTML 有載入，但 **JavaScript 沒跑完** 或 **載入失敗**。

- **請做**：
  1. 按 **F12** 開開發者工具 → 點 **Console** 分頁。
  2. 重新整理頁面，看有沒有**紅色錯誤**。
  3. 再點 **Network** 分頁，重新整理，檢查：
     - **`/`**（第一個請求）：狀態應為 **200**。
     - **`/src/main.tsx`**（或類似的 .tsx/.js 請求）：狀態應為 **200**，不能是 404 或 failed。
- **若 `/src/main.tsx` 是 404 或 failed**：代表 Vite 沒正確提供前端，可能是 dev server 沒跑好或重啟後狀態異常，可關掉終端後重新執行 `npm run dev` 再試。

### 3. 一直停在「載入中...」
代表 **`/api/auth/check`** 沒正常回傳或一直沒結束。

- **請做**：
  1. F12 → **Network**，重新整理頁面。
  2. 找到 **`auth/check`** 或 **`/api/auth/check`** 的請求。
  3. 看狀態是 **200** 還是 **failed / 404 / 500**。
- **若是 200**：回應內容應為 `{"authenticated":true,...}` 或 `{"authenticated":false}`，接著應出現登入頁或中控台；若仍是載入中，可能是前端邏輯或 cache，可試 **Ctrl+Shift+R** 強制重新整理或用無痕視窗。
- **若是 failed / 紅字**：代表連不到後端（網址錯、port 錯、或 server 沒跑），請確認終端有 `serving on port 5001` 且無錯誤。

### 4. 瀏覽器快取
- **請試**：**Ctrl+Shift+R**（強制重新整理）或用**無痕視窗**開 `http://localhost:5001`，看 APP 是否出現。

### 5. 確認目前跑的是最新程式
- 終端若在改程式**之前**就開了，請**關掉終端裡的 npm run dev**，再在專案根目錄執行一次 **`npm run dev`**，等出現 `serving on port 5001` 後再開網頁。

---

## 可選修正：讓其他裝置也能連（改聽 0.0.0.0）

若你是用**手機或另一台電腦**開瀏覽器，目前預設是連不到的。若要開放同一網路其他裝置連線，需要把 server 改成監聽 `0.0.0.0`（已在下方說明如何改）。

**注意**：改完後同一網路的人只要知道你的 IP 和 port 就能連到後台，僅建議在信任的內網使用。

---

## 請回覆這三點，方便進一步判斷

1. 你開網頁的網址是？（例如 `http://localhost:5001`）
2. 畫面是：**完全白**、**一直載入中**、還是**有登入頁但登入後沒東西**？
3. F12 → Console 裡有沒有**紅色錯誤**？有的話請貼一兩行訊息。
